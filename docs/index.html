📚 Documentação COMEX

Documentação completa do sistema de processamento de dados de comércio exterior

Bem-vindo à Documentação COMEX

Esta documentação cobre todos os módulos e processos do sistema COMEX, incluindo metodologias antigas e novas, processamento de dados, projeções e integrações. Use o índice abaixo para navegar entre as diferentes seções.

🏛️ OM - Metodologia Antiga

Processamento de dados de comércio exterior usando a metodologia antiga. Inclui pipelines para múltiplos países com arquitetura em camadas (Raw → Silver → Gold).

Metodologia Antiga Pipeline

Países: ARG, BRA, COL, DEU, EUR, IND, MEX, USA

🌍 NM/globinho

Processamento COMEX Globe - Nova Metodologia. Processa dados históricos de comércio exterior, faz merge com indicadores e envia para API de Cost Drivers.

Nova Metodologia Globe

Países: ARG, BRA, COL, EUR, USA

📊 NM/serie_temporal

Processamento de séries temporais - Nova Metodologia. Aplica técnicas estatísticas avançadas para análise e tratamento de dados de comércio exterior.

Nova Metodologia Série Temporal

Países: BRA, EUR, AUT, BEL, BGR, DEU, FRA, HUN, ITA, NLD, PRT

📈 Projeção CIF

Processo de projeção de valores CIF para dados do Brasil. Calcula valores CIF faltantes com base na projeção do FOB usando razões históricas.

Projeção SQL

País: Brasil (BRA)

🚀 Guia Rápido de Uso

Resumo do guia completo disponível em COMO_USAR_O_PROJETO.md. Importante: Sempre execute os scripts a partir da raiz do projeto usando python -m.

📋 Pré-requisitos

  • Python 3.8+
  • pip install -r requirements.txt
  • Variáveis de ambiente configuradas

🆕 Nova Metodologia (NM)

python -m NM.serie_temporal.COMEX_BRA.COMEX_BRA_NM python -m NM.globinho.ARG.COMEX_ARG_GLOBE

🔄 Metodologia Antiga (OM)

python -m OM.COMEX_BRA.COMEX_BRA python -m OM.COMEX_ARG.COMEX_ARG

🎯 Orquestrador Europeu

python -m NM.serie_temporal.orquestrador_europa_NM

Processa: AUT, ITA, FRA, BEL, DEU, PRT, HUN, BGR, NLD

🔧 Variáveis de Ambiente Obrigatórias

  • COSTDRIVERS_PASSWORD - Senha para autenticação
  • COSTDRIVERS_API_KEY - Chave da API
  • COSTDRIVERS_API_EMAIL - Email para autenticação
  • COSTDRIVERS_API_PASSWORD - Senha da API

Recomendação: Crie um arquivo .env na raiz do projeto com essas variáveis.

💡 Dicas Importantes

  • Sempre execute da raiz: Use python -m a partir da raiz do projeto
  • Verifique variáveis: Muitos erros são causados por variáveis não configuradas
  • Use logs: Os scripts geram logs detalhados para debug
  • Teste localmente: Execute scripts localmente antes de usar em produção

📖 Para mais detalhes, consulte: COMO_USAR_O_PROJETO.md na raiz do projeto
