Documentação: COMEX Índia

1. Visão Geral

O crawler COMEX_IND é responsável por coletar, processar e enviar dados de comércio exterior da Índia. Este pipeline faz parte do framework COMEX e utiliza dados oficiais disponibilizados pelo portal Trade Statistics do Ministério do Comércio e Indústria da Índia através de uma interface web que requer interação com formulários HTML.

O objetivo principal é coletar dados históricos e atuais de importação e exportação indiana desde janeiro de 2020, normalizá-los conforme o padrão do framework e integrá-los ao sistema de cost drivers para análise e projeção de custos.

Características Principais

  • Coleta dados diretamente do portal oficial do governo indiano
  • Utiliza web scraping com BeautifulSoup para interagir com formulários
  • Faz requisições POST com tokens CSRF
  • Coleta dados por país e por mês
  • Processa peso e valor separadamente e depois faz merge
  • Implementa retry automático com biblioteca tenacity
  • Usa sessão requests para manter estado (cookies/tokens)
  • Coleta desde 2020-01-01 até o mês atual
  • Processa múltiplos países automaticamente por mês

3. Fonte de Dados

3.1 Portal Oficial

Os dados são coletados diretamente do portal oficial do Ministério do Comércio e Indústria da Índia:

URLs Base

Importação: https://tradestat.commerce.gov.in/meidb/country_wise_all_commodities_import

Exportação: https://tradestat.commerce.gov.in/meidb/country_wise_all_commodities_export

Este portal disponibiliza dados de comércio exterior através de uma interface web que requer interação com formulários HTML. Os dados são apresentados em tabelas HTML que precisam ser parseadas.

3.2 Tipo de Fonte

  • Tipo: Portal Web com formulários HTML interativos
  • Método de acesso: Web Scraping (BeautifulSoup) + Requisições POST
  • Formato dos dados: Tabelas HTML (parseadas com pd.read_html)
  • Autenticação: Não requerida, mas usa tokens CSRF
  • Frequência de atualização: Mensal (conforme disponibilização pelo governo)

3.3 Estrutura dos Dados

Os dados são coletados através de múltiplas requisições:

  1. Lista de Países: Primeiro obtém lista de países disponíveis para o mês/ano
  2. Dados por País: Para cada país, faz 2 requisições:
    • Uma para obter dados de Peso (Quantity)
    • Outra para obter dados de Valor (USD)
  3. Merge: Faz merge dos dados de peso e valor por NCM, Data, Tipo e País
Nota: O portal requer tokens CSRF (_token) que são extraídos do HTML antes de cada requisição POST. O crawler usa uma sessão requests para manter cookies e estado entre requisições.

4. Processo de Coleta

4.1 Método `collect_import_data()`

Coleta dados de importação da Índia desde 2020-01-01 até o mês atual.

Fluxo:

  1. Gera range de datas mensais desde 2020-01-01 até hoje
  2. Para cada mês/ano:
    • Chama import_data() para coletar dados do mês
    • Adiciona flag ImportExport = 1
    • Adiciona ao conjunto de DataFrames
  3. Concatena todos os DataFrames
  4. Retorna DataFrame consolidado

Tratamento de Erros:

  • Se erro em um mês específico: loga erro e continua com próximo mês
  • Se nenhum dado coletado: retorna DataFrame vazio

4.2 Método `collect_export_data()`

Coleta dados de exportação da Índia desde 2020-01-01 até o mês atual.

Fluxo:

  1. Gera range de datas mensais desde 2020-01-01 até hoje
  2. Para cada mês/ano:
    • Chama export_data() para coletar dados do mês
    • Adiciona flag ImportExport = 0
    • Adiciona ao conjunto de DataFrames
  3. Concatena todos os DataFrames
  4. Retorna DataFrame consolidado

4.3 Método Auxiliar: `safe_request()`

Wrapper resiliente para requisições GET/POST com retry automático usando biblioteca tenacity.

Características:

  • Retry Automático: Até 3 tentativas com backoff exponencial
  • Timeout: 60 segundos por requisição
  • Tratamento de Erros: Captura RequestException, ConnectionError, Timeout
  • Backoff: Espera exponencial (1s, 2s, 4s, até 8s máximo)
  • Logging: Loga cada tentativa de retry

Decorator:

@retry(
    retry=retry_if_exception_type((RequestException, ConnectionError, Timeout)),
    wait=wait_exponential(multiplier=1, min=1, max=8),
    stop=stop_after_attempt(3),
    reraise=True
)
def safe_request(self, s, method, url, **kwargs):
    """Wrapper resiliente para GET/POST com tentativas automáticas."""
    if method.lower() == "get":
        r = s.get(url, timeout=60, **kwargs)
    else:
        r = s.post(url, timeout=60, **kwargs)
    r.raise_for_status()
    return r

4.4 Método Auxiliar: `list_available_countries()`

Obtém lista de países disponíveis para um mês/ano específico através de requisição POST.

Processo:

  1. Faz GET na URL para obter token CSRF
  2. Extrai token do HTML usando BeautifulSoup
  3. Faz POST com dados do mês/ano
  4. Parseia resposta HTML para encontrar tabela e select de países
  5. Filtra países com valor > 0 na tabela
  6. Retorna dicionário {código: nome_do_país}

Parâmetros:

  • tipo: 'import' ou 'export'
  • prefix: Prefixo do formulário ('cwim' para import, 'cwe' para export)
  • mes: Mês (1-12)
  • ano: Ano (ex: 2024)

4.5 Método Auxiliar: `treat_dataframe()`

Padroniza e transforma DataFrame de valores/peso para formato longo (melt).

Processo:

  1. Identifica coluna de NCM (HSCode ou segunda coluna)
  2. Remove colunas duplicadas (regex para detectar duplicatas)
  3. Identifica colunas de datas (formato: MMM-YYYY, ex: "Jan-2024")
  4. Faz melt para transformar formato largo em formato longo
  5. Converte DataRaw (MMM-YYYY) para datetime
  6. Limpa valores numéricos (remove vírgulas, trata "-" como 0)
  7. Adiciona colunas Tipo e Pais
  8. Retorna DataFrame com colunas: ["NCM", value_name, "Data", "Tipo", "Pais"]

Formato de Entrada:

NCM | Jan-2024 | Feb-2024 | Mar-2024 | ...
123  | 1000     | 2000     | 1500     | ...

Formato de Saída:

NCM | Valor/Peso | Data       | Tipo   | Pais
123 | 1000       | 2024-01-01 | import | Brazil
123 | 2000       | 2024-02-01 | import | Brazil

4.6 Método Auxiliar: `import_data()`

Coleta dados de importação para um mês/ano específico.

Processo:

  1. Faz GET na URL para obter token CSRF
  2. Obtém lista de países disponíveis usando list_available_countries()
  3. Para cada país:
    • Faz POST para obter dados de Peso (ReportVal=2)
    • Faz POST para obter dados de Valor (ReportVal=1)
    • Processa cada DataFrame usando treat_dataframe()
    • Faz merge de peso e valor por NCM, Data, Tipo, Pais
    • Adiciona ao conjunto de DataFrames
  4. Concatena todos os DataFrames de países
  5. Retorna DataFrame consolidado

Parâmetros do POST:

  • _token: Token CSRF extraído do HTML
  • cwcimMonth: Mês
  • cwcimYear: Ano
  • cwcimallcount: Código do país
  • cwcimCommodityLevel: "8" (nível de commodity)
  • cwcimReportVal: "1" para valor, "2" para peso
  • cwcimReportYear: "2"

4.7 Método Auxiliar: `export_data()`

Coleta dados de exportação para um mês/ano específico. Similar a import_data() mas com parâmetros diferentes do formulário.

Diferenças em relação a import_data():

  • Usa prefixo "cwe" ao invés de "cwim"
  • Parâmetros do POST usam prefixo "cwcex":
    • cwcexddMonth, cwcexddYear
    • cwcexallcount
    • cwcexddCommodityLevel
    • cwcexddReportVal
    • cwcexddReportYear

Particularidades da Coleta

  • Coleta desde 2020: Processa todos os meses desde 2020-01-01 até o mês atual
  • Web Scraping Interativo: Interage com formulários HTML, não apenas faz download
  • Tokens CSRF: Extrai e usa tokens CSRF para cada requisição POST
  • Sessão Requests: Usa requests.Session() para manter cookies e estado
  • Retry Automático: Implementa retry com backoff exponencial usando tenacity
  • Coleta por País: Coleta dados de múltiplos países por mês
  • Peso e Valor Separados: Faz 2 requisições por país (peso e valor) e depois merge
  • Transformação Melt: Converte formato largo (colunas por mês) para formato longo
  • Headers HTTP Completos: Usa headers detalhados para simular navegador
  • Processamento Mensal: Processa mês a mês sequencialmente

6. Tratamento Específico

6.1 Método `_country_specific_treatment()`

O método implementa tratamento específico para dados da Índia após a normalização de colunas.

O que faz:

  1. Validação de Data:
    • Converte coluna Data para datetime se ainda não estiver
    • Remove linhas com data inválida
  2. Limpeza de NCM:
    • Garante que NCM seja string e aplica trim
    • Remove NCM nulos ou vazios
  3. Limpeza de Valores Numéricos:
    • Remove vírgulas de valores (formato indiano: "1,234.56")
    • Trata "-" como 0
    • Trata "nan" como 0
    • Converte para float e preenche NaN com 0

Código:

def _country_specific_treatment(self, df: pd.DataFrame) -> pd.DataFrame:
    """Tratamento específico para dados da Índia."""
    if df.empty:
        return df
    
    # Converter Data para datetime
    if 'Data' in df.columns:
        df['Data'] = pd.to_datetime(df['Data'], errors='coerce')
        df = df[df['Data'].notna()]
    
    # Garantir que NCM seja string e trim
    if 'ncm' in df.columns:
        df['ncm'] = df['ncm'].astype(str).str.strip()
        df = df[df['ncm'].notna()]
        df = df[df['ncm'] != '']
    
    # Limpar valores numéricos
    if 'valor' in df.columns:
        df['valor'] = (
            df['valor'].astype(str)
            .str.replace(",", "", regex=False)
            .replace("-", "0")
            .replace("nan", "0")
            .astype(float)
            .fillna(0)
        )
    
    if 'peso' in df.columns:
        df['peso'] = (
            df['peso'].astype(str)
            .str.replace(",", "", regex=False)
            .replace("-", "0")
            .replace("nan", "0")
            .astype(float)
            .fillna(0)
        )
    
    return df

6.2 Conversões de Dados

  • Formato de Data: Criada a partir de formato "MMM-YYYY" (ex: "Jan-2024"), convertida para datetime usando pd.to_datetime()
  • Formato Numérico: Remove vírgulas (formato indiano), trata "-" como 0
  • NCM: Mantido como string com trim aplicado
  • Valores em USD: Valores estão em USD (diferente de EUR que usa EUR)

6.3 Filtros Aplicados

  • Filtro de Países: Coleta apenas países com valor > 0 na tabela de países disponíveis
  • Filtro de NCM: Remove NCM nulos ou vazios
  • Filtro de Data: Remove linhas com data inválida

Headers HTTP Detalhados

O crawler usa headers HTTP muito detalhados para simular um navegador Chrome real, incluindo:

  • User-Agent completo do Chrome
  • Headers de segurança (Sec-Fetch-*)
  • Accept-Language
  • Referer e Origin
  • Cache-Control

Isso é necessário porque o portal pode bloquear requisições que não parecem vir de um navegador real.

10. Particularidades e Observações

Pontos de Atenção

  • Tempo de Execução: A coleta pode levar muito tempo pois processa todos os meses desde 2020 e múltiplos países por mês. Cada país requer 2 requisições (peso + valor).
  • Tokens CSRF: O portal requer tokens CSRF que são extraídos do HTML. Se estrutura HTML mudar, pode quebrar a extração de tokens.
  • Web Scraping Frágil: Depende da estrutura HTML do portal. Se estrutura mudar, métodos podem precisar atualização.
  • Múltiplas Requisições: Para cada mês, faz muitas requisições (1 para lista de países + 2 por país). Pode ser lento.
  • Retry Logic: Implementa retry automático, mas se portal estiver indisponível por muito tempo, pode falhar após 3 tentativas.
  • Formato de Dados: Dados vêm em formato largo (colunas por mês) e são transformados para formato longo.
  • Período Fixo: Sempre coleta desde 2020-01-01, independente do parâmetro start_date.
  • Headers HTTP: Usa headers muito detalhados. Se portal mudar validação, pode precisar ajuste.

Diferenças em Relação a Outros Crawlers

  • Web Scraping Interativo: Único que interage com formulários HTML ao invés de apenas fazer download
  • Tokens CSRF: Extrai e usa tokens CSRF para cada requisição
  • Sessão Requests: Usa sessão para manter estado entre requisições
  • Retry com Tenacity: Usa biblioteca tenacity para retry automático com backoff exponencial
  • Coleta por País: Coleta dados de múltiplos países por mês
  • Peso e Valor Separados: Faz 2 requisições por país e depois merge
  • Transformação Melt: Converte formato largo para formato longo
  • Headers Detalhados: Usa headers muito completos para simular navegador
  • pais_name: Usa nome do país (string) ao invés de código (int)
  • Valores em USD: Valores estão em USD (diferente de EUR)

Problemas Conhecidos e Soluções

  • Problema: Erro ao extrair token CSRF
    Causa: Estrutura HTML mudou ou portal indisponível
    Solução: Verificar estrutura HTML atual, ajustar código de extração se necessário
  • Problema: Timeout em requisições
    Causa: Portal lento ou sobrecarregado
    Solução: Retry automático tenta novamente. Se persistir, pode aumentar timeout
  • Problema: Nenhum país encontrado para determinado mês
    Causa: Dados ainda não disponíveis ou portal mudou
    Solução: Crawler continua com próximo mês e loga warning

11. Troubleshooting

11.1 Problemas Comuns

Erro: "Token CSRF não encontrado"

Possíveis Causas:

  • Estrutura HTML do portal mudou
  • Portal temporariamente indisponível
  • Campo de token mudou de nome

Soluções:

  • Verificar manualmente o HTML do portal para encontrar campo _token
  • Atualizar código de extração se necessário
  • Verificar logs para ver resposta HTML recebida

Erro: "Nenhum país encontrado"

Possíveis Causas:

  • Dados ainda não disponíveis para o mês
  • Estrutura da tabela HTML mudou
  • Select de países não encontrado

Soluções:

  • Verificar manualmente se dados estão disponíveis no portal
  • Verificar estrutura HTML atual
  • Ajustar método list_available_countries() se necessário

Erro: "Timeout" ou "Connection Error"

Possíveis Causas:

  • Portal lento ou sobrecarregado
  • Problemas de rede
  • Portal bloqueando requisições

Soluções:

  • Retry automático tenta até 3 vezes com backoff exponencial
  • Verificar conectividade com o portal
  • Verificar se headers HTTP estão corretos
  • Pode ser necessário adicionar delay entre requisições

Erro: "Tabela não encontrada" ao parsear HTML

Possíveis Causas:

  • Estrutura HTML mudou
  • Portal retornou erro em vez de tabela
  • Dados não disponíveis

Soluções:

  • Verificar resposta HTML recebida
  • Verificar se tabela existe na resposta
  • Ajustar parsing se estrutura mudou
