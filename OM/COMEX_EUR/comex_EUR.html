Documentação: COMEX Europa

1. Visão Geral

O crawler COMEX_EUR é responsável por coletar, processar e enviar dados de comércio exterior de todos os países europeus através do portal Eurostat (Statistical Office of the European Union). Este pipeline é fundamental pois fornece dados agregados de toda a Europa, que são posteriormente utilizados por outros crawlers (como COMEX_DEU) para extrair dados de países específicos.

O objetivo principal é coletar dados históricos e atuais de importação e exportação de todos os países europeus, normalizá-los conforme o padrão do framework e disponibilizá-los no Azure ADLS2 para uso por outros pipelines.

Características Principais

  • Coleta dados de todos os países europeus de uma vez
  • Utiliza portal oficial Eurostat
  • Processa arquivos .7z comprimidos
  • Extrai arquivos .dat de dentro dos .7z
  • Coleta import e export juntos e depois separa
  • Processa apenas os 3 arquivos mais recentes
  • Usa web scraping para descobrir arquivos disponíveis
  • Gera dados que são reutilizados por outros crawlers (ex: COMEX_DEU)

3. Fonte de Dados

3.1 Portal Oficial

Os dados são coletados diretamente do portal oficial do Eurostat (Statistical Office of the European Union):

URL Base

Portal: https://ec.europa.eu/eurostat/api/dissemination/files/?sort=1&dir=comext%2FCOMEXT_DATA%2FPRODUCTS

Este portal disponibiliza arquivos .7z comprimidos contendo dados de comércio exterior de todos os países europeus. Cada arquivo contém dados mensais completos (import + export) de todos os países.

3.2 Tipo de Fonte

  • Tipo: Portal Web com arquivos .7z para download
  • Método de acesso: Web Scraping (BeautifulSoup) + Download HTTP
  • Formato dos dados: .7z (comprimido) contendo .dat (CSV-like)
  • Encoding: ISO-8859-1 (Latin-1)
  • Separador CSV: Vírgula (,)
  • Autenticação: Não requerida (dados públicos)
  • Frequência de atualização: Mensal (conforme disponibilização pelo Eurostat)

3.3 Estrutura dos Arquivos

Os arquivos seguem um padrão de nomenclatura:

  • Formato: full_v2_YYYYMM.7z
  • Exemplo: full_v2_202412.7z (dados de dezembro de 2024)
  • Conteúdo: Arquivo .dat com nome similar: full_YYYYMM.dat

Cada arquivo .7z contém um arquivo .dat com dados completos de importação e exportação de todos os países europeus.

Nota: O crawler processa apenas os 3 arquivos mais recentes para otimizar tempo e recursos. Arquivos com "52" no nome (dados anuais) são excluídos automaticamente.

4. Processo de Coleta

4.1 Método `collect()`

Método principal de coleta. Este método sobrescreve o comportamento padrão do framework para coletar import e export juntos e depois separar.

Fluxo:

  1. Verifica cache local (se disponível e válido)
  2. Se não houver cache, chama _collect_all_data()
  3. Separa dados em import e export usando coluna TRADE_TYPE ou FLOW
  4. Salva separadamente no ADLS como import_raw.parquet e export_raw.parquet
  5. Retorna tupla (import_df, export_df)

Separação Import/Export:

  • Se TRADE_TYPE existe: 'I' = importação, 'E' = exportação
  • Se FLOW existe: '1' = importação, '2' = exportação

4.2 Método `collect_import_data()` e `collect_export_data()`

Métodos mantidos para compatibilidade, mas não são usados diretamente. A coleta é feita através do método collect() que coleta tudo junto.

Estes métodos retornam DataFrame vazio e são mantidos apenas para compatibilidade com a interface do framework.

4.3 Método Auxiliar: `_get_available_files()`

Descobre automaticamente quais arquivos .7z estão disponíveis no portal Eurostat usando web scraping.

Processo:

  1. Acessa URL base do portal
  2. Faz parsing do HTML usando BeautifulSoup
  3. Encontra tabela HTML com lista de arquivos
  4. Lê tabela usando pd.read_html()
  5. Filtra apenas arquivos .7z (Type == "7z")
  6. Extrai data do nome do arquivo (formato: full_v2_YYYYMM)
  7. Exclui arquivos com "52" no nome (dados anuais)
  8. Filtra arquivos com tamanho > 20 MB
  9. Extrai links de download do HTML
  10. Ordena por data (mais recente primeiro)
  11. Retorna lista de dicionários com informações dos arquivos

Retorno:

[
    {
        "url": "https://ec.europa.eu/...",
        "name": "full_v2_202412.7z",
        "date": datetime(2024, 12, 1),
        "size": 123.4  # MB
    },
    ...
]

4.4 Método Auxiliar: `_download_and_extract()`

Baixa arquivo .7z e extrai arquivo .dat usando biblioteca py7zr.

Processo:

  1. Cria pasta temporária se não existir (.temp/eurostat/)
  2. Verifica se arquivo .dat já existe (cache local)
  3. Se não existir, verifica se .7z existe
  4. Se .7z não existir, baixa do portal (stream=True, timeout=300s)
  5. Salva .7z na pasta temporária
  6. Extrai .7z usando py7zr
  7. Procura arquivo .dat extraído
  8. Se não encontrar com nome esperado, procura qualquer .dat
  9. Retorna caminho do arquivo .dat extraído

Características:

  • Cache Local: Mantém arquivos extraídos para evitar re-extração
  • Download em Stream: Usa stream=True para arquivos grandes
  • Timeout Longo: 300 segundos (5 minutos) para downloads grandes
  • Extraction Inteligente: Procura arquivo .dat mesmo se nome mudou
  • Dependência: Requer biblioteca py7zr

Código:

def _download_and_extract(self, file_info: Dict[str, any]) -> Optional[Path]:
    """Baixa arquivo .7z e extrai arquivo .dat."""
    # Criar pasta temporária
    if not self.temp_folder:
        self._create_temp_folder()
    
    # Verificar se .dat já existe (cache)
    if file_path_dat.exists():
        return file_path_dat
    
    # Baixar .7z se não existir
    if not file_path_7z.exists():
        response = requests.get(file_info["url"], stream=True, timeout=300)
        # Salvar arquivo
    
    # Extrair .dat do .7z
    import py7zr
    with py7zr.SevenZipFile(file_path_7z, mode='r') as archive:
        archive.extractall(path=self.temp_folder)
    
    return file_path_dat

4.5 Método Auxiliar: `_process_dat_file()`

Processa arquivo .dat completo (import + export juntos) e retorna DataFrame.

Processo:

  1. Lê arquivo .dat usando pd.read_csv():
    • Separador: vírgula (,)
    • Encoding: ISO-8859-1
    • Tipo: string inicialmente (dtype=str)
    • low_memory=False para arquivos grandes
  2. Filtra linhas onde PRODUCT_NC != "TOTAL"
  3. Converte PERIOD (YYYYMM) para coluna Data (datetime)
  4. Remove coluna PERIOD
  5. Adiciona colunas frete e seguro com valor 0.0
  6. Retorna DataFrame processado

Nota: Este método processa import e export juntos. A separação é feita posteriormente.

4.6 Método Auxiliar: `_collect_all_data()`

Coleta todos os dados (import + export) processando os 3 arquivos mais recentes.

Processo:

  1. Obtém lista de arquivos disponíveis (já ordenados por data)
  2. Seleciona apenas os 3 arquivos mais recentes
  3. Cria pasta temporária
  4. Para cada arquivo:
    • Baixa e extrai arquivo .dat
    • Processa arquivo .dat completo
    • Adiciona ao conjunto de DataFrames
  5. Concatena todos os DataFrames
  6. Retorna DataFrame consolidado

Tratamento de Erros:

  • Se arquivo não encontrado: loga warning e continua
  • Se erro ao baixar/extrair: loga warning e continua
  • Se erro ao processar: loga warning e continua
  • Se nenhum arquivo processado: retorna DataFrame vazio

4.7 Métodos de Gerenciamento de Pasta Temporária

`_create_temp_folder()`:

  • Cria pasta .temp/eurostat/ se não existir
  • Armazena caminho em self.temp_folder

`_cleanup_temp_folder()`:

  • Remove pasta temporária após uso
  • Chamado automaticamente no final do pipeline
  • Permite cache entre chamadas durante execução

Particularidades da Coleta

  • Coleta Unificada: Coleta import e export juntos, depois separa
  • 3 Arquivos Mais Recentes: Processa apenas os 3 arquivos mais recentes para otimizar
  • Web Scraping: Descobre arquivos disponíveis automaticamente via HTML parsing
  • Arquivos .7z: Requer extração usando biblioteca py7zr
  • Pasta Temporária: Usa .temp/eurostat/ para extração e cache
  • Cache Local: Mantém arquivos extraídos para evitar re-extração
  • Filtro de Tamanho: Processa apenas arquivos > 20 MB
  • Exclusão de Anuais: Exclui arquivos com "52" no nome (dados anuais)
  • Encoding ISO-8859-1: Usa encoding Latin-1 para dados europeus
  • Valores em EUR: Valores estão em EUR, não USD

5. Mapeamento de Colunas (Data Contract)

5.1 Importação

O mapeamento de colunas para dados de importação:

Coluna Original Coluna Normalizada Tipo Descrição
PRODUCT_NC ncm int Código NCM (Nomenclatura Comum do Mercosul)
REPORTER pais_origem str Código do país de origem (ex: DE, FR, IT)
PARTNER pais_transacao str Código do país parceiro na transação
VALUE_EUR valor float Valor em EUR
QUANTITY_KG peso float Peso líquido em quilogramas
PERIOD Data datetime Período da transação (formato YYYYMM)
TRADE_TYPE ou FLOW ImportExport int Tipo de comércio (1 = importação, 0 = exportação)

Colunas mantidas após normalização:

['Data', 'ncm', 'pais_origem', 'pais_transacao', 'ImportExport', 'peso', 'valor', 'frete', 'seguro']

5.2 Exportação

O mapeamento de colunas para dados de exportação é idêntico ao de importação:

Coluna Original Coluna Normalizada Tipo Descrição
PRODUCT_NC ncm int Código NCM
REPORTER pais_origem str Código do país de origem
PARTNER pais_transacao str Código do país parceiro
VALUE_EUR valor float Valor em EUR
QUANTITY_KG peso float Peso líquido em quilogramas
PERIOD Data datetime Período da transação (formato YYYYMM)

Colunas mantidas após normalização:

['Data', 'ncm', 'pais_origem', 'pais_transacao', 'ImportExport', 'peso', 'valor', 'frete', 'seguro']
Observação: Note que este crawler mantém colunas pais_origem e pais_transacao separadas, permitindo que outros crawlers (como COMEX_DEU) filtrem por país específico.

6. Tratamento Específico

6.1 Método `_country_specific_treatment()`

Este crawler não implementa tratamento específico. Usa o tratamento padrão do framework.

O tratamento específico é feito durante o processamento do arquivo .dat no método _process_dat_file():

  • Filtro de NCM "TOTAL"
  • Conversão de PERIOD para Data
  • Adição de colunas frete e seguro com valor 0

6.2 Conversões de Dados

  • Formato de Data: Criada a partir de PERIOD no formato YYYYMM, convertida para datetime usando pd.to_datetime()
  • Filtro de NCM: Remove registros com PRODUCT_NC == "TOTAL" durante processamento
  • Encoding ISO-8859-1: Usa encoding Latin-1 para dados europeus
  • Separador CSV: Vírgula (,)
  • Valores em EUR: Valores estão em EUR, não USD
  • Frete e Seguro: Adicionados com valor 0.0 (não disponíveis nos dados do Eurostat)

6.3 Filtros Aplicados

  • Filtro de Arquivo: Processa apenas arquivos > 20 MB
  • Filtro de Data: Exclui arquivos com "52" no nome (dados anuais)
  • Filtro de Quantidade: Processa apenas os 3 arquivos mais recentes
  • Filtro de NCM: Remove PRODUCT_NC == "TOTAL"

10. Método: atualizar_historico_europa()

Objetivo

Atualiza os arquivos históricos de cada país europeu com dados novos coletados. Este método deve ser chamado após collect() ter sido executado.

10.1 Fluxo de Execução

atualizar_historico_europa()
    ↓
1. Verificar se raw_import_df e raw_export_df existem
    ↓
2. Concatenar import + export em df_historic
    ↓
3. Carregar histórico de referência (BGR)
    ↓ (NM/dados/BGR/database/historical.parquet)
4. Comparar PERIOD máximo
    ↓ (Se max_period_new <= max_period_db → Early Exit)
5. Filtrar por países de interesse (REPORTER)
    ↓ (AT, BE, BG, DE, FR, HU, IT, NL, PT)
6. Para cada país:
    ↓
    a. Carregar histórico existente
    ↓ (NM/dados/{ISO3}/database/historical.parquet)
    b. Normalizar TRADE_TYPE (I/E → 1/2)
    ↓
    c. Concatenar novo + existente
    ↓
    d. Normalizar tipos (PERIOD, VALUE_EUR, QUANTITY_KG)
    ↓
    e. Salvar parquet atualizado
    ↓
7. Log resumo final

11. Particularidades e Observações

Pontos de Atenção

  • Dependência de py7zr: Biblioteca obrigatória que pode não estar instalada por padrão. Instalar com pip install py7zr
  • Arquivos Grandes: Arquivos .7z podem ser muito grandes (centenas de MB). Downloads podem levar vários minutos.
  • Pasta Temporária: Cria pasta .temp/eurostat/ que pode ocupar espaço significativo. É limpa automaticamente ao final.
  • Web Scraping: Depende da estrutura HTML do portal. Se estrutura mudar, _get_available_files() pode precisar atualização.
  • 3 Arquivos Mais Recentes: Processa apenas 3 arquivos. Se precisar de mais dados históricos, ajustar código.
  • Separação Import/Export: Dados vêm juntos e são separados após coleta. Requer coluna TRADE_TYPE ou FLOW.
  • Valores em EUR: Valores estão em EUR, não USD. Considerar em análises comparativas.
  • Dados para Outros Crawlers: Este crawler gera dados que são usados por COMEX_DEU e outros.

Diferenças em Relação a Outros Crawlers

  • Coleta Unificada: Único crawler que coleta import e export juntos e depois separa
  • Arquivos .7z: Único que processa arquivos .7z comprimidos
  • Web Scraping Avançado: Faz parsing de tabela HTML para descobrir arquivos
  • Múltiplos Países: Coleta dados de todos os países europeus de uma vez
  • Dependência de py7zr: Requer biblioteca específica para extração
  • Pasta Temporária: Usa pasta temporária para extração e cache
  • 3 Arquivos: Processa apenas 3 arquivos mais recentes (otimização)
  • Geração de Dados Derivados: Seus dados são reutilizados por outros crawlers
  • Valores em EUR: Usa EUR ao invés de USD
  • Colunas Especiais: Mantém pais_origem e pais_transacao para filtragem
  • Atualização de Históricos: Possui método dedicado para atualizar históricos de 9 países europeus

Problemas Conhecidos e Soluções

  • Problema: Erro "py7zr não encontrado"
    Solução: Instalar biblioteca: pip install py7zr
  • Problema: Timeout em downloads grandes
    Solução: Timeout já é de 300s. Se persistir, verificar conexão ou aumentar timeout
  • Problema: Pasta temporária não é limpa
    Solução: Limpeza automática no final do pipeline. Se falhar, limpar manualmente .temp/eurostat/
  • Problema: Nenhum arquivo encontrado
    Solução: Verificar se estrutura HTML do portal mudou. Verificar logs para detalhes

12. Troubleshooting

12.1 Problemas Comuns

Erro: "py7zr não encontrado"

Causa: Biblioteca py7zr não está instalada

Solução:

pip install py7zr

Ou adicionar ao requirements.txt do projeto.

Erro: "Tabela não encontrada no portal Eurostat"

Possíveis Causas:

  • Estrutura HTML do portal mudou
  • Portal temporariamente indisponível
  • URL do portal mudou

Soluções:

  • Verificar manualmente o portal para confirmar estrutura atual
  • Atualizar método _get_available_files() se necessário
  • Verificar logs para ver resposta HTTP recebida

Erro: "Arquivo .dat não encontrado após extração"

Possíveis Causas:

  • Nome do arquivo .dat mudou
  • Estrutura do .7z mudou
  • Arquivo .7z corrompido

Soluções:

  • O código já procura por qualquer arquivo .dat se não encontrar com nome esperado
  • Verificar conteúdo do .7z manualmente
  • Verificar logs para ver quais arquivos foram extraídos

Erro: "Coluna TRADE_TYPE ou FLOW não encontrada"

Possíveis Causas:

  • Estrutura dos dados mudou
  • Formato do arquivo .dat mudou

Soluções:

  • Verificar estrutura dos dados coletados
  • Verificar quais colunas existem no DataFrame
  • Ajustar método collect() se necessário
