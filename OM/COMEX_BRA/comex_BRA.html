Documentação: COMEX Brasil

1. Visão Geral

O crawler COMEX_BRA é responsável por coletar, processar e enviar dados de comércio exterior do Brasil. Este pipeline faz parte do framework COMEX e utiliza dados oficiais disponibilizados pelo portal do governo brasileiro através do Ministério da Economia (Produtividade e Comércio Exterior).

O objetivo principal é coletar dados históricos e atuais de importação e exportação brasileira, normalizá-los conforme o padrão do framework e integrá-los ao sistema de cost drivers para análise e projeção de custos.

Características Principais

  • Coleta dados diretamente do portal oficial do governo brasileiro
  • Processa arquivos CSV disponibilizados publicamente
  • Suporta dados de importação e exportação
  • Processa múltiplos anos automaticamente
  • Utiliza web scraping para descobrir arquivos disponíveis

3. Fonte de Dados

3.1 Portal Oficial

Os dados são coletados diretamente do portal oficial do governo brasileiro:

URL Base

Portal: https://www.gov.br/produtividade-e-comercio-exterior/pt-br/assuntos/comercio-exterior/estatisticas/base-de-dados-bruta

Este portal disponibiliza arquivos CSV com dados brutos de comércio exterior organizados por ano e tipo (importação/exportação).

3.2 Tipo de Fonte

  • Tipo: Portal Web com arquivos CSV para download
  • Método de acesso: Web Scraping (BeautifulSoup) + Download HTTP
  • Formato dos dados: CSV delimitado por ponto e vírgula (;)
  • Encoding: UTF-8
  • Autenticação: Não requerida (dados públicos)
  • Frequência de atualização: Mensal (conforme disponibilização pelo governo)

3.3 Estrutura dos Arquivos

Os arquivos CSV são nomeados seguindo o padrão:

  • Importação: Contém "ncm" e "import" no nome do arquivo
  • Exportação: Contém "ncm" e "EXP" no nome do arquivo
  • Ano: O ano aparece no nome do arquivo
Nota: O crawler utiliza web scraping para descobrir automaticamente quais arquivos estão disponíveis no portal, filtrando por ano e tipo (importação/exportação).

4. Processo de Coleta

4.1 Método `collect_import_data()`

Coleta dados de importação do Brasil através do método auxiliar _process_ie_type("import").

Fluxo:

  1. Chama _process_ie_type("import")
  2. O método auxiliar descobre arquivos disponíveis
  3. Filtra apenas arquivos de importação
  4. Baixa e processa cada arquivo
  5. Concatena todos os DataFrames em um único resultado

4.2 Método `collect_export_data()`

Coleta dados de exportação do Brasil através do método auxiliar _process_ie_type("export").

Fluxo:

  1. Chama _process_ie_type("export")
  2. O método auxiliar descobre arquivos disponíveis
  3. Filtra apenas arquivos de exportação
  4. Baixa e processa cada arquivo
  5. Concatena todos os DataFrames em um único resultado

4.3 Método Auxiliar: `_get_available_files()`

Descobre automaticamente quais arquivos CSV estão disponíveis no portal.

Funcionamento:

  1. Acessa a URL base do portal
  2. Faz parsing do HTML usando BeautifulSoup
  3. Procura por links que contenham:
    • "ncm" no nome
    • "csv" no nome
    • Ano dentro do range configurado
  4. Identifica o tipo (import/export) baseado na presença de "EXP" no nome
  5. Retorna lista de dicionários com informações dos arquivos

Retorno:

[
    {
        "url": "https://...",
        "ImportExport": "import" ou "export",
        "year": "2024"
    },
    ...
]

4.4 Método Auxiliar: `_download_file()`

Baixa um arquivo CSV específico do portal.

Características:

  • Usa requests.get() com stream=True para downloads grandes
  • SSL não verificado (verify=False) devido a problemas de certificado no portal
  • Timeout de 60 segundos
  • Retorna bytes do arquivo ou None em caso de erro
  • Logging detalhado de cada operação

4.5 Método Auxiliar: `_process_ie_type()`

Processa dados de importação ou exportação de forma unificada.

Processo:

  1. Obtém lista de arquivos disponíveis
  2. Filtra arquivos do tipo solicitado (import/export)
  3. Para cada arquivo:
    • Baixa o arquivo CSV
    • Lê o CSV usando pandas (separador: ;, encoding: UTF-8)
    • Cria coluna Data a partir de CO_MES e CO_ANO
    • Adiciona flag ImportExport (1 para import, 0 para export)
    • Adiciona ao conjunto de DataFrames
  4. Concatena todos os DataFrames em um único resultado
  5. Retorna DataFrame consolidado

Tratamento de Erros:

  • Se arquivo não encontrado: loga warning e continua
  • Se erro ao baixar: loga erro e continua com próximo arquivo
  • Se erro ao processar CSV: loga erro e continua
  • Se nenhum arquivo encontrado: retorna DataFrame vazio

Particularidades da Coleta

  • Processamento Multi-Ano: Processa automaticamente todos os anos desde start_date até o ano atual
  • Descoberta Automática: Não precisa conhecer URLs específicas, descobre arquivos disponíveis automaticamente
  • Web Scraping: Utiliza BeautifulSoup para fazer parsing do HTML do portal
  • SSL Não Verificado: Configura SSL não verificado devido a problemas de certificado no portal
  • Encoding UTF-8: Todos os arquivos são lidos com encoding UTF-8
  • Separador CSV: Usa ponto e vírgula (;) como separador
  • Criação de Data: Combina CO_MES e CO_ANO para criar coluna Data

5. Mapeamento de Colunas (Data Contract)

5.1 Importação

O mapeamento de colunas para dados de importação:

Coluna Original Coluna Normalizada Tipo Descrição
CO_NCM ncm int Código NCM (Nomenclatura Comum do Mercosul)
CO_PAIS pais_id int Código do país de origem
KG_LIQUIDO peso float Peso líquido em quilogramas
VL_FOB valor float Valor FOB em USD
VL_FRETE frete float Valor do frete em USD
VL_SEGURO seguro float Valor do seguro em USD
CO_MES + CO_ANO Data datetime Data da transação (criada durante processamento)

Colunas mantidas após normalização:

['Data', 'ncm', 'ImportExport', 'pais_id', 'peso', 'valor', 'frete', 'seguro']

5.2 Exportação

O mapeamento de colunas para dados de exportação:

Coluna Original Coluna Normalizada Tipo Descrição
CO_NCM ncm int Código NCM (Nomenclatura Comum do Mercosul)
CO_PAIS pais_id int Código do país de destino
KG_LIQUIDO peso float Peso líquido em quilogramas
VL_FOB valor float Valor FOB em USD
CO_MES + CO_ANO Data datetime Data da transação (criada durante processamento)

Colunas mantidas após normalização:

['Data', 'ncm', 'ImportExport', 'pais_id', 'peso', 'valor']
Observação: A exportação não possui colunas de frete e seguro, pois esses custos são incorridos apenas na importação.

6. Tratamento Específico

6.1 Método `_country_specific_treatment()`

O método implementa tratamento específico para dados do Brasil após a normalização de colunas.

O que faz:

  1. Remove colunas auxiliares: Remove as colunas CO_MES e CO_ANO que foram usadas apenas para criar a coluna Data e não são mais necessárias
  2. Valida coluna Data: Verifica se a coluna Data existe após normalização (deve existir, mas valida por segurança)

Por que é necessário:

  • As colunas CO_MES e CO_ANO são usadas apenas durante o processamento para criar a coluna Data
  • Após a criação da coluna Data, essas colunas não são mais necessárias e devem ser removidas
  • Isso mantém o DataFrame limpo e evita confusão

Código:

def _country_specific_treatment(self, df: pd.DataFrame) -> pd.DataFrame:
    """Tratamento específico para dados do Brasil."""
    # Remover colunas que não são mais necessárias após normalização
    columns_to_drop = ['CO_MES', 'CO_ANO']
    for col in columns_to_drop:
        if col in df.columns:
            df.drop(columns=[col], inplace=True)
    
    # Garantir que coluna Data existe
    if 'Data' not in df.columns:
        self.logger.warning("Coluna Data não encontrada após normalização")
    
    return df

6.2 Conversões de Dados

  • Formato de Data: Criada a partir de CO_MES e CO_ANO no formato MM-YYYY, convertida para datetime usando pd.to_datetime()
  • Formato Numérico: Dados são lidos como string inicialmente (dtype=str) e convertidos posteriormente pelo framework conforme schema do data-contract
  • Encoding: UTF-8 (padrão)
  • Separador CSV: Ponto e vírgula (;)

6.3 Filtros Aplicados

Não há filtros específicos aplicados aos dados brasileiros. Todos os dados coletados são processados.

7. Configurações e Parâmetros

7.1 Parâmetros do `__init__()`

Parâmetro Tipo Padrão Descrição
start_date datetime ou None None Data inicial para coleta. Se None, usa padrão do framework (primeiro dia do mês há 60 dias)
use_azure bool True Se True, usa Azure ADLS2 para storage; se False, usa storage local

7.2 Configurações Específicas

  • URL Base: https://www.gov.br/produtividade-e-comercio-exterior/pt-br/assuntos/comercio-exterior/estatisticas/base-de-dados-bruta
  • SSL: Configurado para não verificar certificados (ssl._create_unverified_context)
  • Timeout: 30 segundos para requisições HTTP (portal), 60 segundos para downloads
  • Data Contract Path: include/data-contract.yaml
  • IDs Table Path: include/IDS_comex.xlsx

Aviso: SSL Não Verificado

O crawler configura SSL para não verificar certificados devido a problemas conhecidos com o certificado SSL do portal do governo. Isso é necessário para permitir o acesso aos dados, mas deve ser considerado ao avaliar a segurança da conexão.

10. Particularidades e Observações

Pontos de Atenção

  • Web Scraping: O crawler depende da estrutura HTML do portal. Se a estrutura mudar, o método _get_available_files() pode precisar ser atualizado.
  • SSL Não Verificado: O crawler desabilita verificação SSL devido a problemas com o certificado do portal. Isso é necessário mas deve ser considerado em ambientes de produção.
  • Nomes de Arquivos: O crawler identifica arquivos por padrões no nome. Se o padrão de nomenclatura mudar, pode ser necessário ajustar a lógica de filtragem.
  • Disponibilidade: Os arquivos são disponibilizados mensalmente pelo governo. O crawler processa todos os anos disponíveis desde start_date.
  • Encoding: Todos os arquivos são assumidos como UTF-8. Se houver arquivos com encoding diferente, pode ser necessário ajustar.

Diferenças em Relação a Outros Crawlers

  • Descoberta Automática: Ao contrário de outros crawlers que usam URLs fixas, este crawler descobre automaticamente quais arquivos estão disponíveis.
  • Web Scraping: Utiliza BeautifulSoup para fazer parsing do HTML, enquanto outros crawlers podem usar APIs diretas.
  • Processamento Multi-Arquivo: Processa múltiplos arquivos CSV automaticamente, um para cada ano.
  • SSL Não Verificado: É o único crawler que desabilita verificação SSL explicitamente.
  • Criação de Data: Cria coluna Data a partir de duas colunas separadas (CO_MES e CO_ANO), enquanto outros podem receber data já formatada.

Problemas Conhecidos e Soluções

  • Problema: Erro de certificado SSL ao acessar portal
    Solução: SSL não verificado configurado no __init__()
  • Problema: Avisos do BeautifulSoup sobre parsing HTML
    Solução: Avisos desabilitados com warnings.filterwarnings()
  • Problema: Arquivos não encontrados para determinado ano
    Solução: Crawler continua processando outros anos e loga warning

11. Troubleshooting

11.1 Problemas Comuns

Erro na Coleta: "Nenhum arquivo encontrado"

Possíveis Causas:

  • Estrutura HTML do portal mudou
  • Padrão de nomes de arquivos mudou
  • Portal temporariamente indisponível
  • Range de anos muito restrito

Soluções:

  • Verificar manualmente o portal para confirmar estrutura atual
  • Atualizar método _get_available_files() se necessário
  • Verificar logs para ver quantos arquivos foram encontrados
  • Ajustar start_date se necessário

Erro ao Baixar Arquivo: Timeout ou Erro de Conexão

Possíveis Causas:

  • Arquivo muito grande
  • Portal lento ou sobrecarregado
  • Problemas de rede

Soluções:

  • Aumentar timeout em _download_file()
  • Verificar conectividade com o portal
  • Tentar baixar manualmente para validar disponibilidade

Erro no Tratamento: "Coluna Data não encontrada"

Possíveis Causas:

  • Colunas CO_MES ou CO_ANO não existem nos dados
  • Formato dos dados mudou
  • Erro na criação da coluna Data

Soluções:

  • Verificar estrutura dos dados coletados
  • Verificar se CO_MES e CO_ANO existem
  • Verificar logs para ver onde falhou
