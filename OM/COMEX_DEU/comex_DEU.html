Documentação: COMEX Alemanha

1. Visão Geral

O crawler COMEX_DEU é responsável por processar dados de comércio exterior da Alemanha. Este pipeline é único no framework COMEX pois não coleta dados diretamente de uma fonte externa. Em vez disso, ele processa dados já coletados do agregado EUR (Europa) que estão armazenados no Azure ADLS2, filtrando especificamente os registros da Alemanha (país código 'DE').

O objetivo principal é extrair dados alemães do conjunto maior de dados europeus, normalizá-los conforme o padrão do framework e integrá-los ao sistema de cost drivers para análise e projeção de custos.

⚠️ Dependência Crítica

Este crawler depende completamente dos dados do EUR. Os dados do EUR devem ser coletados e processados primeiro antes de executar este crawler. Sem dados do EUR no ADLS, este crawler não funcionará.

Características Principais

  • Não realiza coleta direta de dados externos
  • Carrega dados do EUR já coletados do Azure ADLS2
  • Filtra registros por pais_origem == 'DE'
  • Reutiliza dados já processados, economizando recursos
  • Dependente do pipeline EUR para funcionar

3. Fonte de Dados

3.1 Origem dos Dados

Os dados não são coletados diretamente de uma fonte externa. Em vez disso, são carregados do Azure ADLS2 a partir dos dados já coletados pelo pipeline EUR.

Caminho no ADLS

Importação: staging/comex/EUR/raw/import_raw.parquet

Exportação: staging/comex/EUR/raw/export_raw.parquet

Estes arquivos são gerados pelo pipeline COMEX_EUR que coleta dados de todos os países europeus.

3.2 Tipo de Fonte

  • Tipo: Dados derivados (filtrados do EUR)
  • Método de acesso: Carregamento de arquivos Parquet do Azure ADLS2
  • Formato dos dados: Parquet (Apache Parquet)
  • Filtro aplicado: pais_origem == 'DE' ou REPORTER == 'DE'
  • Dependência: Pipeline EUR deve ser executado primeiro
Nota: O pipeline EUR coleta dados de todos os países europeus e armazena no ADLS. Este crawler simplesmente filtra os dados alemães desse conjunto maior.

3.3 Estrutura dos Dados

Os dados carregados do EUR contêm informações de todos os países europeus. O crawler DEU filtra apenas os registros onde:

  • pais_origem == 'DE' (após normalização), ou
  • REPORTER == 'DE' (antes da normalização)

O código 'DE' é o código ISO 3166-1 alpha-2 para a Alemanha (Deutschland).

4. Processo de Coleta

4.1 Método `collect_import_data()`

Carrega dados de importação da Alemanha a partir dos dados do EUR armazenados no ADLS.

Fluxo:

  1. Carrega arquivo import_raw.parquet do caminho staging/comex/EUR/raw/
  2. Chama _filter_country() para filtrar registros da Alemanha
  3. Retorna DataFrame filtrado com apenas dados alemães

Código:

def collect_import_data(self) -> pd.DataFrame:
    """Coleta dados de importação da Alemanha."""
    self.logger.info("Carregando dados de importação do EUR do ADLS...")
    
    # Carregar dados do EUR do ADLS
    eur_raw_path = "staging/comex/EUR/raw"
    df_eur = self._load_from_storage(eur_raw_path, "import_raw.parquet")
    df_deu = self._filter_country(df_eur, data_type="importação")
    return df_deu

4.2 Método `collect_export_data()`

Carrega dados de exportação da Alemanha a partir dos dados do EUR armazenados no ADLS.

Fluxo:

  1. Carrega arquivo export_raw.parquet do caminho staging/comex/EUR/raw/
  2. Chama _filter_country() para filtrar registros da Alemanha
  3. Retorna DataFrame filtrado com apenas dados alemães

Código:

def collect_export_data(self) -> pd.DataFrame:

4.3 Método Auxiliar: `_filter_country()`

Filtra dados do EUR para manter apenas registros da Alemanha (DE).

Processo:

  1. Verifica se DataFrame está vazio (retorna vazio se sim)
  2. Identifica coluna de país:
    • Procura por pais_origem (após normalização)
    • Se não encontrar, procura por REPORTER (antes da normalização)
    • Se nenhuma for encontrada, retorna DataFrame vazio
  3. Filtra registros onde coluna == 'DE'
  4. Loga resultado do filtro
  5. Retorna DataFrame filtrado

Código:

def _filter_country(self, df_eur: pd.DataFrame, data_type: str = "dados") -> pd.DataFrame:
    """Filtra dados do EUR para manter apenas registros da Alemanha (DE)."""
    if df_eur.empty:
        self.logger.warning(f"Nenhum dado de {data_type} encontrado no EUR")
        return pd.DataFrame()
    
    # Identificar coluna para filtro
    col = None
    if 'pais_origem' in df_eur.columns:
        col = 'pais_origem'
    elif 'REPORTER' in df_eur.columns:
        col = 'REPORTER'
    else:
        self.logger.error("Nenhuma coluna de país encontrada")
        return pd.DataFrame()
    
    # Filtrar por país == 'DE'
    df_deu = df_eur[df_eur[col] == 'DE'].copy()
    
    self.logger.info(f"Dados DEU filtrados: {len(df_deu)} registros (de {len(df_eur)} total)")
    return df_deu

Particularidades da Coleta

  • Sem Coleta Externa: Não faz requisições HTTP ou downloads de arquivos
  • Carregamento do ADLS: Usa método _load_from_storage() do framework
  • Filtro Simples: Aplica filtro por código de país 'DE'
  • Dependência do EUR: Requer que dados do EUR estejam disponíveis no ADLS
  • Eficiência: Reutiliza dados já coletados, evitando coletas duplicadas
  • Flexibilidade de Coluna: Funciona com pais_origem ou REPORTER

6. Tratamento Específico

6.1 Método `_country_specific_treatment()`

O método implementa tratamento específico para dados da Alemanha após a normalização de colunas.

O que faz:

  1. Filtra NCM "TOTAL": Remove linhas onde ncm == "TOTAL" (agregações que não são úteis)
  2. Mapeia TRADE_TYPE: Se existir coluna TRADE_TYPE e não existir ImportExport, mapeia valores:
    • 'I' → 1 (importação)
    • 'E' → 0 (exportação)
    Remove coluna TRADE_TYPE após mapeamento
  3. Remove coluna 'pais': Remove coluna pais se existir (não necessária após normalização)
  4. Valida coluna Data:
    • Se não existir, tenta criar a partir de PERIOD (formato YYYYMM)
    • Se existir, garante que está em formato datetime

Código:

def _country_specific_treatment(self, df: pd.DataFrame) -> pd.DataFrame:
    """Tratamento específico para dados da Alemanha."""
    # Filtrar linhas onde NCM == "TOTAL"
    if 'ncm' in df.columns:
        df = df[df['ncm'] != "TOTAL"].copy()
    
    # Mapear TRADE_TYPE para ImportExport
    if 'TRADE_TYPE' in df.columns and 'ImportExport' not in df.columns:
        IE_map = {'I': 1, 'E': 0}
        df['ImportExport'] = df['TRADE_TYPE'].map(IE_map)
        df.drop(columns=['TRADE_TYPE'], inplace=True)
    
    # Remover coluna Pais se existir
    if 'pais' in df.columns:
        df.drop(columns=['pais'], inplace=True)
    
    # Garantir que coluna Data existe e está no formato correto
    if 'Data' not in df.columns:
        if 'PERIOD' in df.columns:
            df['Data'] = pd.to_datetime(df['PERIOD'].astype(str), format='%Y%m', errors='coerce')
    else:
        if not pd.api.types.is_datetime64_any_dtype(df['Data']):
            df['Data'] = pd.to_datetime(df['Data'], errors='coerce')
    
    return df

6.2 Conversões de Dados

  • Formato de Data: Criada a partir de PERIOD no formato YYYYMM, convertida para datetime usando pd.to_datetime()
  • Filtro de NCM: Remove registros com ncm == "TOTAL" (agregações)
  • Mapeamento TRADE_TYPE: Converte 'I'/'E' para 1/0 (ImportExport)
  • Valores em EUR: Os valores estão em EUR, não em USD como outros países

6.3 Filtros Aplicados

  • Filtro Principal: pais_origem == 'DE' ou REPORTER == 'DE' (aplicado na coleta)
  • Filtro de NCM: Remove ncm == "TOTAL" (aplicado no tratamento específico)

10. Particularidades e Observações

Pontos de Atenção Críticos

  • Dependência do EUR: Este crawler não funciona sem dados do EUR no ADLS. O pipeline EUR deve ser executado primeiro.
  • Azure ADLS2 Obrigatório: Não funciona com storage local, pois precisa acessar dados do EUR que estão no ADLS.
  • Ordem de Execução: Execute sempre EUR antes de DEU. Se EUR falhar, DEU também falhará.
  • Filtro por Coluna: O código procura por pais_origem ou REPORTER. Se nenhuma existir, retorna DataFrame vazio.
  • Valores em EUR: Os valores estão em EUR, não em USD como outros países. Isso pode afetar análises comparativas.

Diferenças em Relação a Outros Crawlers

  • Sem Coleta Externa: Único crawler que não faz requisições HTTP ou downloads
  • Dados Derivados: Processa dados já coletados, não coleta novos dados
  • Dependência Explícita: Depende de outro pipeline (EUR) para funcionar
  • Filtro Simples: A lógica principal é apenas filtrar por código de país
  • Reutilização: Economiza recursos ao reutilizar dados já coletados
  • Colunas Especiais: Mantém pais_origem e pais_transacao (não apenas pais_id)
  • Valores em EUR: Usa EUR ao invés de USD

Problemas Conhecidos e Soluções

  • Problema: DataFrame vazio após coleta
    Causa: Dados do EUR não disponíveis ou filtro não encontrou registros da Alemanha
    Solução: Verificar se pipeline EUR foi executado e se há dados da Alemanha no conjunto EUR
  • Problema: Erro ao carregar do ADLS
    Causa: Permissões insuficientes ou caminho incorreto
    Solução: Verificar permissões no Azure ADLS2 e caminho staging/comex/EUR/raw/
  • Problema: Coluna de país não encontrada
    Causa: Estrutura dos dados do EUR mudou
    Solução: Verificar se dados do EUR contêm pais_origem ou REPORTER

11. Troubleshooting

11.1 Problemas Comuns

Erro: "Nenhum dado encontrado no EUR"

Possíveis Causas:

  • Pipeline EUR não foi executado
  • Dados do EUR não foram salvos no ADLS
  • Caminho incorreto no ADLS
  • Permissões insuficientes para acessar ADLS

Soluções:

  • Execute o pipeline EUR primeiro: COMEX_EUR().run()
  • Verifique se arquivos existem em staging/comex/EUR/raw/
  • Verifique permissões no Azure ADLS2
  • Verifique logs do pipeline EUR para erros

Erro: "Nenhum registro da Alemanha encontrado após filtro"

Possíveis Causas:

  • Dados do EUR não contêm registros da Alemanha
  • Coluna de país tem nome diferente
  • Código do país não é 'DE' nos dados

Soluções:

  • Verificar se dados do EUR contêm registros com REPORTER == 'DE'
  • Verificar estrutura dos dados do EUR
  • Verificar logs para ver quantos registros foram encontrados antes do filtro
  • Inspecionar dados do EUR diretamente

Erro: "Nenhuma coluna de país encontrada"

Possíveis Causas:

  • Estrutura dos dados do EUR mudou
  • Colunas não foram normalizadas corretamente
  • Nome da coluna é diferente

Soluções:

  • Verificar estrutura dos dados do EUR carregados
  • Verificar se normalização está funcionando corretamente
  • Ajustar método _filter_country() se necessário
