Documentação: COMEX Estados Unidos

1. Visão Geral

O crawler COMEX_USA é responsável por coletar, processar e enviar dados de comércio exterior dos Estados Unidos. Este pipeline faz parte do framework COMEX e utiliza dados oficiais disponibilizados pela API U.S. Census Bureau através de endpoints REST que fornecem dados históricos e atuais de importação e exportação.

O objetivo principal é coletar dados históricos de comércio exterior dos EUA (últimos 5 anos), filtrar por países específicos, normalizá-los conforme o padrão do framework e integrá-los ao sistema de cost drivers para análise e projeção de custos.

Características Principais

  • Coleta dados diretamente da API oficial U.S. Census Bureau
  • Utiliza requisições assíncronas para otimizar performance
  • Requer arquivo Excel com mapeamento de NCMs (NCM_IE_USITC.xlsx)
  • Coleta dados dos últimos 5 anos (ajustado recentemente)
  • Exclui automaticamente os últimos 2 meses (dados podem estar incompletos)
  • Filtra por lista específica de países (USA_COMEX_PAISES)
  • Para importações: calcula frete + seguro a partir de valor_cif
  • Usa API key para autenticação

3. Fonte de Dados

3.1 API Oficial

Os dados são coletados diretamente da API oficial do U.S. Census Bureau:

URLs da API

Importação: https://api.census.gov/data/timeseries/intltrade/imports/hs

Exportação: https://api.census.gov/data/timeseries/intltrade/exports/hs

Esta API fornece dados históricos e atuais de comércio exterior dos Estados Unidos em formato JSON. Requer API key para autenticação e suporta filtros por commodity (NCM), país, e período.

3.2 Tipo de Fonte

  • Tipo: REST API (JSON)
  • Método de acesso: Requisições HTTP GET assíncronas
  • Formato dos dados: JSON (convertido para DataFrame)
  • Autenticação: API Key (parâmetro key)
  • Frequência de atualização: Mensal (conforme disponibilização pelo Census Bureau)
  • Documentação: Exports Variables | Imports Variables

3.3 Estrutura dos Dados

A API retorna dados em formato JSON com arrays. Cada requisição retorna dados para um NCM específico e um mês específico. O crawler faz múltiplas requisições assíncronas (uma por NCM) e depois consolida os resultados.

Parâmetros da API:

  • get: Campos a retornar (ex: "I_COMMODITY,CTY_CODE,CTY_NAME,GEN_VAL_MO,GEN_CIF_MO,GEN_QY1_MO")
  • time: Período no formato YYYY-MM
  • I_COMMODITY ou E_COMMODITY: Código NCM (HS10)
  • COMM_LVL: Nível de commodity ("HS10")
  • key: API Key
Nota: A API requer um arquivo Excel (NCM_IE_USITC.xlsx) que contém o mapeamento de NCMs e seus tipos (Import/Export). Este arquivo é usado para determinar quais NCMs coletar e como classificá-los.

4. Processo de Coleta

4.1 Método `collect_import_data()`

Coleta dados de importação dos Estados Unidos para todos os meses no range de datas.

Fluxo:

  1. Itera sobre cada mês no date_range
  2. Para cada mês, chama _fetch_month()
  3. Extrai DataFrames de importação do resultado
  4. Concatena todos os DataFrames
  5. Retorna DataFrame consolidado

4.2 Método `collect_export_data()`

Coleta dados de exportação dos Estados Unidos para todos os meses no range de datas.

Fluxo:

  1. Itera sobre cada mês no date_range
  2. Para cada mês, chama _fetch_month()
  3. Extrai DataFrames de exportação do resultado
  4. Concatena todos os DataFrames
  5. Retorna DataFrame consolidado

4.3 Método Auxiliar: `_load_ncm_file()`

Carrega arquivo Excel com mapeamento de NCMs e tipos de comércio.

Arquivo: NCM_IE_USITC.xlsx

Estrutura Esperada:

  • Coluna IDIndicePrincipal: ID do índice principal
  • Coluna IE: Tipo de comércio (1 = Import, 0 = Export)
  • Coluna NCM: Código NCM (HS10)

Tratamento de Erros:

  • Se arquivo não encontrado: loga erro e retorna DataFrame vazio
  • Se erro ao ler: loga erro e retorna DataFrame vazio

4.4 Método Auxiliar: `_build_date_range()`

Constrói range de datas para coleta (formato YYYY-MM), excluindo os últimos 2 meses.

Processo:

  1. Gera range de datas mensais desde start_period até hoje
  2. Exclui os últimos 2 meses (dados podem estar incompletos)
  3. Converte para formato YYYY-MM
  4. Retorna lista de strings

Exemplo:

# Se hoje é 2024-12-15
# start_period = 2019-12-01 (5 anos atrás)
# date_range = ['2019-12', '2020-01', ..., '2024-09', '2024-10']
# (exclui novembro e dezembro de 2024)

4.5 Método Auxiliar: `_mount_requests_for_month()`

Monta lista de requisições para um mês específico, uma requisição por NCM.

Processo:

  1. Filtra NCMs do arquivo Excel por tipo (IE: 1 = Import, 0 = Export)
  2. Para cada NCM filtrado:
    • Monta parâmetros da API específicos para import ou export
    • Cria dicionário com URL, método, parâmetros e NCM
    • Adiciona à lista de requisições
  3. Retorna lista de requisições

Parâmetros para Importação:

{
    "get": "I_COMMODITY,CTY_CODE,CTY_NAME,GEN_VAL_MO,GEN_CIF_MO,GEN_QY1_MO",
    "time": "2024-01",
    "I_COMMODITY": "1234567890",
    "COMM_LVL": "HS10",
    "key": "0b09d9c5b84851e0fa3a1995cf041f0a8ac3b380"
}

Parâmetros para Exportação:

{
    "get": "E_COMMODITY,CTY_CODE,CTY_NAME,ALL_VAL_MO,QTY_1_MO",
    "time": "2024-01",
    "E_COMMODITY": "1234567890",
    "COMM_LVL": "HS10",
    "key": "0b09d9c5b84851e0fa3a1995cf041f0a8ac3b380"
}

4.6 Método Auxiliar: `_fetch_month()`

Coleta dados de um mês específico (import e export) usando requisições assíncronas.

Processo:

  1. Monta requisições para import e export usando _mount_requests_for_month()
  2. Executa requisições assíncronas usando ApiAsync:
    • Uma chamada para importações
    • Uma chamada para exportações
  3. Converte respostas JSON em DataFrames
  4. Retorna dicionário com listas de DataFrames

Uso de ApiAsync:

dados_import = ApiAsync(True, lista=reqs_import).run()
dados_export = ApiAsync(True, lista=reqs_export).run()

ApiAsync executa múltiplas requisições em paralelo, otimizando o tempo de coleta.

Particularidades da Coleta

  • Requisições Assíncronas: Usa ApiAsync para executar múltiplas requisições em paralelo
  • Arquivo NCM Obrigatório: Requer arquivo NCM_IE_USITC.xlsx para determinar quais NCMs coletar
  • Coleta por NCM: Uma requisição por NCM por mês
  • Período de 5 Anos: Coleta dados dos últimos 5 anos (ajustado recentemente)
  • Exclusão de Últimos 2 Meses: Exclui automaticamente para evitar dados incompletos
  • API Key: Usa API key hardcoded no código (pode precisar atualização)
  • Filtro de Países: Filtra por lista específica de países após coleta
  • Valor CIF para Importações: API retorna GEN_CIF_MO que é mapeado para valor_cif

5. Mapeamento de Colunas (Data Contract)

5.1 Importação

O mapeamento de colunas para dados de importação:

CTY_CODE pais_id int Código do país (U.S. Census)
CTY_NAME pais_name str Nome do país

GEN_CIF_MO valor_cif float Valor CIF em USD
GEN_QY1_MO peso float Quantidade em quilogramas

Colunas mantidas após normalização:

['Data', 'ncm', 'ImportExport', 'pais_id', 'pais_name', 'peso', 'valor', 'valor_cif']
Nota Importante: Para importações, a API retorna valor_cif diretamente. No tratamento específico, este valor é usado para calcular frete + seguro = valor_cif - valor, e então a coluna valor_cif é removida antes de seguir para as próximas fases.

O mapeamento de colunas para dados de exportação:

CTY_CODE pais_id int Código do país (U.S. Census)
CTY_NAME pais_name str Nome do país

Colunas mantidas após normalização:

['Data', 'ncm', 'ImportExport', 'pais_id', 'pais_name', 'peso', 'valor', 'frete', 'seguro']
Nota: Para exportações, não há valor CIF. As colunas frete e seguro são adicionadas com valor 0.0 no tratamento específico.

6. Tratamento Específico

6.1 Método `_country_specific_treatment()`

O método implementa tratamento específico para dados dos Estados Unidos após a normalização de colunas.

O que faz:

  1. Filtro de Países:
    • Filtra dados por lista USA_COMEX_PAISES
    • Mantém apenas países que estão na lista
  2. Conversão de Tipos:
    • Converte peso e valor para float
  3. Adição de Colunas Frete e Seguro:
    • Adiciona frete e seguro com valor 0.0 se não existirem
  4. Validação de Data:
    • Garante que coluna Data está em formato datetime
    • Converte de formato YYYY-MM se necessário

Código:

def _country_specific_treatment(self, df: pd.DataFrame) -> pd.DataFrame:
    """Tratamento específico para dados dos Estados Unidos."""
    if df.empty:
        return df
    
    # Filtrar por lista de países
    if 'pais_name' in df.columns:
        df = df[df['pais_name'].isin(self.USA_COMEX_PAISES)].copy()
        self.logger.info(f"Dados filtrados por países: {len(df)} registros")
    
    # Converter para float
    if 'peso' in df.columns and 'valor' in df.columns:
        df['peso'] = df['peso'].astype(float)
        df['valor'] = df['valor'].astype(float)
    
    # Adicionar colunas frete e seguro
    if 'frete' not in df.columns:
        df['frete'] = 0.0
    if 'seguro' not in df.columns:
        df['seguro'] = 0.0
    
    # Garantir que Data está em formato datetime
    if 'Data' in df.columns:
        if not pd.api.types.is_datetime64_any_dtype(df['Data']):
            df['Data'] = pd.to_datetime(df['Data'], format='%Y-%m', errors='coerce')
    
    return df

⚠️ Tratamento de valor_cif para Importações

Importante: O código atual não trata explicitamente valor_cif no método _country_specific_treatment(). No entanto, conforme documentado anteriormente, o tratamento de valor_cif para importações deveria:

  1. Calcular frete_seguro = valor_cif - valor
  2. Garantir que frete_seguro >= 0
  3. Atribuir frete = frete_seguro e seguro = 0.0
  4. Remover coluna valor_cif (não faz parte do schema final)

Este tratamento pode estar sendo feito em outra parte do código ou pode precisar ser adicionado.

6.2 Conversões de Dados

  • Formato de Data: Criada a partir de formato YYYY-MM da API, convertida para datetime usando pd.to_datetime()
  • Filtro de Países: Mantém apenas países da lista USA_COMEX_PAISES
  • Valores em USD: Valores estão em USD
  • NCM: Código HS10 (10 dígitos)

6.3 Filtros Aplicados

  • Filtro de Países: Apenas países da lista USA_COMEX_PAISES (mais de 100 países)
  • Filtro de Período: Exclui últimos 2 meses automaticamente

10. Particularidades e Observações

Pontos de Atenção

  • Arquivo NCM Obrigatório: O crawler não funciona sem o arquivo NCM_IE_USITC.xlsx. Este arquivo deve estar presente no diretório raiz.
  • API Key Hardcoded: A API key está hardcoded no código. Se expirar ou mudar, precisa ser atualizada manualmente.
  • Múltiplas Requisições: Para cada mês, faz uma requisição por NCM. Se houver muitos NCMs, pode gerar muitas requisições.
  • Requisições Assíncronas: Usa ApiAsync para otimizar, mas ainda pode levar tempo dependendo do número de NCMs e meses.
  • Exclusão de Últimos 2 Meses: Sempre exclui últimos 2 meses para evitar dados incompletos. Isso é automático e não pode ser desabilitado facilmente.
  • Filtro de Países: Filtra por lista específica de mais de 100 países. Se precisar adicionar/remover países, editar lista USA_COMEX_PAISES.
  • Valor CIF: Para importações, API retorna valor_cif. O tratamento atual não processa explicitamente este valor para calcular frete e seguro separadamente.
  • Período de 5 Anos: Coleta dados dos últimos 5 anos por padrão. Isso foi ajustado recentemente (antes era 3 meses).

Diferenças em Relação a Outros Crawlers

  • API REST: Único que usa API REST oficial ao invés de web scraping ou arquivos
  • Requisições Assíncronas: Usa ApiAsync para otimizar múltiplas requisições
  • Arquivo NCM Externo: Requer arquivo Excel externo para mapeamento de NCMs
  • API Key: Usa autenticação via API key
  • Valor CIF Direto: API retorna valor_cif diretamente para importações
  • Filtro de Países Extenso: Filtra por lista muito grande de países (100+)
  • Exclusão Automática: Exclui automaticamente últimos 2 meses
  • Coleta por NCM: Uma requisição por NCM, não por país
  • Período Longo: Coleta 5 anos de dados por padrão

Problemas Conhecidos e Soluções

  • Problema: Erro "Arquivo NCM_IE_USITC.xlsx não encontrado"
    Causa: Arquivo não está no diretório raiz ou caminho incorreto
    Solução: Verificar se arquivo existe em ./NCM_IE_USITC.xlsx ou ajustar caminho
  • Problema: Erro de autenticação na API
    Causa: API key expirada ou inválida
    Solução: Obter nova API key do U.S. Census Bureau e atualizar no código
  • Problema: Timeout em requisições
    Causa: Muitas requisições ou API lenta
    Solução: ApiAsync já otimiza. Se persistir, pode adicionar delay entre requisições

11. Troubleshooting

11.1 Problemas Comuns

Erro: "Arquivo NCM_IE_USITC.xlsx não encontrado"

Possíveis Causas:

  • Arquivo não está no diretório raiz
  • Caminho incorreto
  • Nome do arquivo diferente

Soluções:

  • Verificar se arquivo existe em ./NCM_IE_USITC.xlsx
  • Verificar estrutura do arquivo (deve ter colunas: IDIndicePrincipal, IE, NCM)
  • Ajustar caminho em self.ncm_file_path se necessário

Erro: "API Key inválida" ou "401 Unauthorized"

Possíveis Causas:

  • API key expirada
  • API key incorreta
  • Limite de requisições excedido

Soluções:

  • Obter nova API key em: Census API Key Signup
  • Atualizar self.API_KEY no código
  • Verificar limites de rate limiting da API

Erro: "Nenhum NCM encontrado"

Possíveis Causas:

  • Arquivo NCM não tem NCMs para o tipo (IE) especificado
  • Arquivo NCM está vazio
  • Filtro IE está incorreto

Soluções:

  • Verificar conteúdo do arquivo NCM_IE_USITC.xlsx
  • Verificar se coluna IE tem valores 0 (export) e 1 (import)
  • Verificar se coluna NCM tem valores válidos

Erro: "Nenhum dado coletado"

Possíveis Causas:

  • API não retornou dados para os NCMs/meses solicitados
  • Erro em requisições assíncronas
  • Filtro de países removeu todos os dados

Soluções:

  • Verificar logs para ver se requisições foram bem-sucedidas
  • Testar requisição manual para um NCM específico
  • Verificar se países retornados estão na lista USA_COMEX_PAISES

  • pandas - Manipulação de dados
  • costdrivers - Módulo com classe ApiAsync para requisições assíncronas
  • openpyxl ou xlrd - Para ler arquivo Excel NCM_IE_USITC.xlsx

⚠️ Instalação de Dependências

Certifique-se de instalar todas as dependências antes de executar: 

pip install pandas openpyxl

O módulo costdrivers com a classe ApiAsync deve estar disponível no projeto.
