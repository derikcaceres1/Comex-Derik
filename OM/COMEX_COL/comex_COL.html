Documentação: COMEX Colômbia

1. Visão Geral

O crawler COMEX_COL é responsável por coletar, processar e enviar dados de comércio exterior da Colômbia. Este pipeline faz parte do framework COMEX e utiliza dados oficiais disponibilizados pelo DANE (Departamento Administrativo Nacional de Estadística) através de seu portal de microdados, que disponibiliza arquivos ZIP contendo dados históricos e atuais de importação e exportação.

O objetivo principal é coletar dados históricos e atuais de comércio exterior colombiano desde um ano inicial configurável (padrão: ano atual - 1), normalizá-los conforme o padrão do framework e integrá-los ao sistema de cost drivers para análise e projeção de custos.

Características Principais

  • Coleta dados diretamente do portal oficial DANE
  • Utiliza web scraping com BeautifulSoup para extrair links de download
  • Processa arquivos ZIP aninhados (ZIP dentro de ZIP)
  • Coleta dados recentes (ano atual) e históricos (desde ano inicial configurável)
  • Suporta configuração de ano inicial via parâmetro (padrão: ano atual - 1)
  • Processa múltiplos arquivos CSV dentro dos ZIPs
  • Trata problemas de certificado SSL do servidor DANE

3. Fonte de Dados

3.1 Portal Oficial

Os dados são coletados diretamente do portal oficial do DANE (Departamento Administrativo Nacional de Estadística):

URLs

Importação Recente: https://microdatos.dane.gov.co/index.php/catalog/856/get-microdata

Exportação Recente: https://microdatos.dane.gov.co/index.php/catalog/859/get-microdata

Importação Histórica: https://microdatos.dane.gov.co/index.php/catalog/473/get-microdata

Exportação Histórica: https://microdatos.dane.gov.co/index.php/catalog/472/get-microdata

O portal disponibiliza dados através de páginas HTML que contêm links de download para arquivos ZIP. Os dados recentes são disponibilizados em uma única página, enquanto os dados históricos são disponibilizados em uma página com múltiplos links (um por ano).

3.2 Tipo de Fonte

  • Tipo: Portal Web com links de download para arquivos ZIP
  • Método de acesso: Web Scraping (BeautifulSoup) + Download HTTP
  • Formato dos dados: ZIP contendo CSV (pode ter ZIPs aninhados)
  • Encoding: ISO-8859-1 (Latin-1)
  • Separador CSV: Ponto e vírgula (;)
  • Autenticação: Não requerida (dados públicos)
  • SSL: Requer verify=False devido a problemas de certificado
  • Frequência de atualização: Mensal (conforme disponibilização pelo DANE)

3.3 Estrutura dos Arquivos

Os arquivos seguem uma estrutura hierárquica:

  1. ZIP Principal: Contém um ou mais arquivos
  2. ZIPs Aninhados: Podem conter ZIPs dentro do ZIP principal
  3. CSVs: Arquivos CSV dentro dos ZIPs (aninhados ou diretos)

Cada arquivo CSV contém dados mensais, com a primeira coluna representando o período no formato YYMM (ex: "2401" para janeiro de 2024).

⚠️ Problema de Certificado SSL

O servidor DANE apresenta problemas com certificado SSL. Por isso, todas as requisições HTTP usam verify=False e os avisos SSL são suprimidos usando urllib3.disable_warnings().

4. Processo de Coleta

4.1 Método `collect_import_data()`

Coleta dados de importação da Colômbia (recentes + históricos).

Fluxo:

  1. Coleta Dados Recentes:
    • Obtém link de download da URL de importação recente
    • Baixa e processa arquivo ZIP
    • Adiciona flag ImportExport = 1
  2. Coleta Dados Históricos:
    • Obtém todos os links históricos desde start_year até ano atual
    • Para cada ano encontrado:
      • Baixa e processa arquivo ZIP do ano
      • Adiciona flag ImportExport = 1
      • Adiciona ao conjunto de DataFrames
  3. Consolidação:
    • Concatena todos os DataFrames (recentes + históricos)
    • Retorna DataFrame consolidado

4.2 Método `collect_export_data()`

Coleta dados de exportação da Colômbia (recentes + históricos). Segue o mesmo fluxo de collect_import_data(), mas usa URLs de exportação e adiciona flag ImportExport = 0.

4.3 Método Auxiliar: `_get_download_link()`

Obtém o link de download de uma página HTML (para URLs com um único arquivo).

Processo:

  1. Faz requisição GET na URL (com verify=False)
  2. Faz parsing do HTML usando BeautifulSoup
  3. Encontra todos os inputs com type="image"
  4. Extrai link do atributo onclick do primeiro input
  5. Procura por strings contendo "download" no onclick
  6. Retorna o primeiro link encontrado

Código:

def _get_download_link(self, source_url: str) -> str:
    """Obtém o link de download da página HTML."""
    response = requests.get(source_url, timeout=30, verify=False)
    soup = BeautifulSoup(response.text, 'html.parser')
    inputs = soup.find_all('input', type="image")
    onclick = inputs[0].get('onclick', '')
    links = [link.strip() for link in onclick.split("'") if 'download' in link]
    return links[0] if links else None

4.4 Método Auxiliar: `_get_historical_download_links()`

Obtém todos os links de download da página HTML histórica, filtrando por anos necessários.

Processo:

  1. Faz requisição GET na URL histórica (com verify=False)
  2. Faz parsing do HTML usando BeautifulSoup
  3. Encontra todos os inputs com type="image"
  4. Determina prefixo esperado:
    • "Impo_" para importação
    • "Expo_" para exportação
  5. Calcula anos necessários: desde start_year até ano atual
  6. Para cada input:
    • Extrai link do onclick
    • Tenta extrair ano do título, onclick ou span próximo usando regex
    • Se ano está no range necessário e ainda não foi adicionado, adiciona ao dicionário
  7. Retorna lista de dicionários com {'year': ano, 'link': link, 'filename': nome}

Extraction de Ano:

O método tenta extrair o ano em três locais diferentes (em ordem de prioridade):

  1. Título do input: Procura padrão como "Impo_2023.zip"
  2. Atributo onclick: Procura padrão no texto do onclick
  3. Span próximo: Procura em span com classe 'resource-info'

Usa regex: rf'{prefix}(\d{{4}})' onde prefix é "Impo_" ou "Expo_".

4.5 Método Auxiliar: `_read_csv_file()`

Lê um arquivo CSV e cria coluna Data a partir da primeira coluna.

Processo:

  1. Lê CSV usando pd.read_csv():
    • Encoding: ISO-8859-1
    • Separador: ponto e vírgula (;)
    • Tipo inicial: string (dtype=str)
    • low_memory=False para arquivos grandes
  2. Remove espaços em branco dos nomes das colunas
  3. Cria coluna Data a partir da primeira coluna (formato YYMM)
  4. Retorna DataFrame processado

4.6 Método Auxiliar: `_read_files_from_link()`

Baixa e processa arquivos ZIP aninhados de um link.

Processo:

  1. Baixa arquivo ZIP do link (com verify=False e stream=True)
  2. Abre ZIP usando ZipFile
  3. Para cada arquivo no ZIP:
    • Se é ZIP aninhado:
      • Abre ZIP aninhado
      • Para cada CSV dentro do ZIP aninhado:
        • Lê CSV usando _read_csv_file()
        • Adiciona à lista de DataFrames
    • Se é CSV direto:
      • Lê CSV usando _read_csv_file()
      • Adiciona à lista de DataFrames
  4. Concatena todos os DataFrames
  5. Retorna DataFrame consolidado

Tratamento de Erros:

  • Se erro ao baixar: loga erro e retorna DataFrame vazio
  • Se erro ao processar: loga erro e retorna DataFrame vazio
  • Se nenhum CSV encontrado: loga warning e retorna DataFrame vazio

Particularidades da Coleta

  • Coleta Dupla: Coleta dados recentes (ano atual) e históricos (desde start_year)
  • Ano Inicial Configurável: Padrão é ano atual - 1, mas pode ser configurado
  • ZIPs Aninhados: Processa ZIPs dentro de ZIPs automaticamente
  • Múltiplos CSVs: Processa todos os CSVs encontrados e concatena
  • Web Scraping: Extrai links de download do HTML usando BeautifulSoup
  • SSL Desabilitado: Usa verify=False devido a problemas de certificado
  • Extraction Inteligente: Extrai ano dos nomes de arquivos usando regex
  • Evita Duplicatas: Usa dicionário para evitar links duplicados por ano
  • Encoding Latin-1: Usa ISO-8859-1 para dados colombianos
  • Formato de Data: Primeira coluna do CSV contém período no formato YYMM

5.1 Importação

O mapeamento de colunas para dados de importação:

O mapeamento de colunas para dados de exportação:

Coluna Original Coluna Normalizada Tipo Descrição

FOBDOL valor float Valor FOB em dólares
PBK peso float Peso líquido em quilogramas

['Data', 'ncm', 'ImportExport', 'peso', 'valor', 'frete', 'seguro']

6. Tratamento Específico

6.1 Método `_country_specific_treatment()`

O método implementa tratamento específico para dados da Colômbia após a normalização de colunas.

O que faz:

  1. Remove NCM Nulos:
    • Remove linhas onde ncm é nulo
  2. Remove Linhas com '-':
    • Remove linhas onde valor contém '-'
    • Remove linhas onde peso contém '-'
    • O caractere '-' indica dados não disponíveis
  3. Converte Formato Numérico:
    • Para valor e peso:
      • Remove pontos (.) usados como separador de milhar
      • Substitui vírgula (,) por ponto (.) para separador decimal
      • Exemplo: "1.234,56" → 1234.56
    • Converte para float
  4. Converte Data:
    • Converte coluna Data de formato YYMM para datetime
    • Usa pd.to_datetime() com formato '%y%m'
    • Remove linhas com data inválida

Código:

def _country_specific_treatment(self, df: pd.DataFrame) -> pd.DataFrame:
    """Tratamento específico para dados da Colômbia."""
    # Remover NCM nulo
    df = df[~df['ncm'].isnull()]
    
    # Remover linhas com '-' em valor e peso
    df = df[~df['valor'].astype(str).str.contains('-', na=False)]
    df = df[~df['peso'].astype(str).str.contains('-', na=False)]
    
    # Converter formato numérico (formato colombiano: 1.234,56)
    df['valor'] = df['valor'].astype(str).apply(
        lambda x: str(x).replace('.', '').replace(',', '.').strip()
    ).astype(float)
    
    df['peso'] = df['peso'].astype(str).apply(
        lambda x: str(x).replace('.', '').replace(',', '.')
    ).astype(float)
    
    # Converter Data para datetime (formato '%y%m')
    df['Data'] = pd.to_datetime(df['Data'], format='%y%m', errors='coerce')
    df = df[df['Data'].notna()]
    
    return df

6.2 Conversões de Dados

  • Formato de Data: Criada a partir da primeira coluna do CSV no formato YYMM (ex: "2401" = janeiro de 2024), convertida para datetime usando pd.to_datetime() com formato '%y%m'
  • Formato Numérico: Formato colombiano (ex: "1.234,56") convertido para formato padrão (1234.56)
  • Encoding: ISO-8859-1 (Latin-1) para dados colombianos
  • Separador CSV: Ponto e vírgula (;)

6.3 Filtros Aplicados

  • Filtro de NCM: Remove linhas com NCM nulo
  • Filtro de Dados Inválidos: Remove linhas com '-' em valor ou peso
  • Filtro de Data: Remove linhas com data inválida após conversão

Tratamento de start_date

O parâmetro start_date é tratado de forma flexível:

  • None: Usa padrão (ano atual - 1)
  • datetime: Extrai o ano do objeto datetime
  • int: Usa diretamente como ano
  • Outros tipos: Tenta converter para int. Se falhar, usa padrão e loga warning

O ano inicial é armazenado em self.start_year e usado para filtrar quais anos históricos coletar.

10. Particularidades e Observações

Pontos de Atenção

  • Problema de SSL: O servidor DANE tem problemas com certificado SSL. Todas as requisições usam verify=False. Isso é necessário mas pode gerar avisos.
  • ZIPs Aninhados: Os arquivos podem conter ZIPs dentro de ZIPs. O código processa isso automaticamente, mas pode ser lento para arquivos grandes.
  • Múltiplos CSVs: Cada ZIP pode conter múltiplos CSVs. Todos são processados e concatenados.
  • Formato Numérico: Dados usam formato colombiano (1.234,56). O tratamento específico converte para formato padrão.
  • Formato de Data: Primeira coluna do CSV contém período no formato YYMM. Precisa ser convertida para datetime.
  • Web Scraping Frágil: Depende da estrutura HTML do portal. Se estrutura mudar, métodos podem precisar atualização.
  • Extraction de Ano: Extrai ano dos nomes de arquivos usando regex. Se formato mudar, pode não funcionar corretamente.
  • Ano Inicial Padrão: Por padrão coleta desde ano atual - 1. Se precisar de mais dados históricos, especificar start_date.

Diferenças em Relação a Outros Crawlers

  • Coleta Dupla: Coleta dados recentes e históricos de fontes diferentes
  • ZIPs Aninhados: Único que processa ZIPs dentro de ZIPs
  • Ano Inicial Configurável: Permite configurar ano inicial para coleta histórica
  • SSL Desabilitado: Requer desabilitar verificação SSL devido a problemas do servidor
  • Formato Numérico Especial: Converte formato colombiano (1.234,56) para padrão
  • Formato de Data YYMM: Usa formato de 4 dígitos (YYMM) ao invés de YYYY-MM
  • Encoding Latin-1: Usa ISO-8859-1 ao invés de UTF-8
  • Separador CSV: Usa ponto e vírgula (;) ao invés de vírgula
  • Extraction Inteligente: Extrai ano de múltiplos locais no HTML

Problemas Conhecidos e Soluções

  • Problema: Erro de SSL ao fazer requisições
    Causa: Certificado SSL inválido no servidor DANE
    Solução: Código já usa verify=False e suprime avisos
  • Problema: Nenhum link de download encontrado
    Causa: Estrutura HTML mudou ou portal indisponível
    Solução: Verificar estrutura HTML atual, ajustar código se necessário
  • Problema: Ano não extraído corretamente
    Causa: Formato do nome do arquivo mudou
    Solução: Verificar formato atual, ajustar regex se necessário

11. Troubleshooting

11.1 Problemas Comuns

Erro: "SSL Certificate verification failed"

Causa: Problema de certificado SSL no servidor DANE

Solução: O código já trata isso usando verify=False. Se ainda aparecer erro, verificar se urllib3.disable_warnings() está sendo chamado.

Erro: "Nenhum input de imagem encontrado na página"

Possíveis Causas:

  • Estrutura HTML do portal mudou
  • Portal temporariamente indisponível
  • URL incorreta

Soluções:

  • Verificar manualmente a estrutura HTML do portal
  • Atualizar método _get_download_link() se necessário
  • Verificar logs para ver resposta HTML recebida

Erro: "Nenhum arquivo CSV encontrado no ZIP"

Possíveis Causas:

  • ZIP não contém CSVs
  • Estrutura do ZIP mudou
  • Arquivo ZIP corrompido

Soluções:

  • Verificar conteúdo do ZIP manualmente
  • Verificar se há ZIPs aninhados que não estão sendo processados
  • Verificar logs para ver quais arquivos foram encontrados

Erro: "Nenhum link histórico encontrado"

Possíveis Causas:

  • Anos solicitados não estão disponíveis
  • Formato dos nomes de arquivos mudou
  • Regex não está encontrando os anos

Soluções:

  • Verificar quais anos estão disponíveis no portal
  • Verificar formato dos nomes de arquivos
  • Ajustar regex se formato mudou
  • Verificar se start_year está correto
