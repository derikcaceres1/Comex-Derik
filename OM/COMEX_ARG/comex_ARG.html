Documentação: COMEX Argentina

1. Visão Geral

O crawler COMEX_ARG é responsável por coletar, processar e enviar dados de comércio exterior da Argentina. Este pipeline faz parte do framework COMEX e utiliza dados oficiais disponibilizados pelo Instituto Nacional de Estadística y Censos (INDEC) da Argentina através de arquivos ZIP contendo dados CSV.

O objetivo principal é coletar dados históricos e atuais de importação e exportação argentina, normalizá-los conforme o padrão do framework e integrá-los ao sistema de cost drivers para análise e projeção de custos.

Características Principais

  • Coleta dados diretamente do portal oficial do INDEC
  • Processa arquivos ZIP contendo dados CSV
  • Suporta dados de importação e exportação
  • Processa múltiplos anos automaticamente (desde 2020)
  • Implementa retry logic para downloads robustos
  • Extrai arquivo específico ou maior arquivo do ZIP

3. Fonte de Dados

3.1 Portal Oficial

Os dados são coletados diretamente do portal oficial do INDEC (Instituto Nacional de Estadística y Censos):

URL Base

Portal: https://comex.indec.gob.ar/files/zips

Este portal disponibiliza arquivos ZIP organizados por ano e tipo (importação/exportação). Cada ZIP contém um ou mais arquivos CSV com dados de comércio exterior.

3.2 Tipo de Fonte

  • Tipo: Arquivos ZIP para download direto via HTTP
  • Método de acesso: Download HTTP GET direto
  • Formato dos dados: ZIP contendo CSV
  • Encoding: Latin-1 (ISO-8859-1)
  • Separador CSV: Ponto e vírgula (;)
  • Autenticação: Não requerida (dados públicos)
  • Frequência de atualização: Mensal (conforme disponibilização pelo INDEC)

3.3 Estrutura dos Arquivos

Os arquivos ZIP seguem um padrão de nomenclatura:

  • Importação: imports_{ANO}_M.zip
  • Exportação: exports_{ANO}_M.zip
  • Exemplo: imports_2024_M.zip, exports_2023_M.zip

Cada ZIP pode conter um ou mais arquivos CSV. O crawler procura primeiro por um arquivo contendo "expopm" no nome. Se não encontrar, seleciona o maior arquivo do ZIP.

Nota: O crawler processa automaticamente todos os anos desde 2020 até o ano atual, gerando URLs dinamicamente baseado no padrão de nomenclatura conhecido.

4. Processo de Coleta

4.1 Método `collect_import_data()`

Coleta dados de importação da Argentina através do método auxiliar _process_ie_type("import").

Fluxo:

  1. Chama _process_ie_type("import")
  2. Gera lista de anos desde 2020 até o ano atual
  3. Para cada ano, constrói URL do ZIP de importação
  4. Baixa e extrai CSV do ZIP
  5. Processa e concatena todos os DataFrames

4.2 Método `collect_export_data()`

Coleta dados de exportação da Argentina através do método auxiliar _process_ie_type("export").

Fluxo:

  1. Chama _process_ie_type("export")
  2. Gera lista de anos desde 2020 até o ano atual
  3. Para cada ano, constrói URL do ZIP de exportação
  4. Baixa e extrai CSV do ZIP
  5. Processa e concatena todos os DataFrames

4.3 Método Auxiliar: `_make_zip_url()`

Constrói a URL do arquivo ZIP baseado no tipo e ano.

Parâmetros:

  • ie_type: 'import' ou 'export'
  • year: Ano como string (ex: '2024')

Formato da URL:

{base_url}/{ie_type}s_{year}_M.zip

Exemplos:
- https://comex.indec.gob.ar/files/zips/imports_2024_M.zip
- https://comex.indec.gob.ar/files/zips/exports_2023_M.zip

4.4 Método Auxiliar: `_download_zip()`

Baixa um arquivo ZIP com lógica de retry robusta.

Características:

  • Retry Logic: Até 3 tentativas com delay de 10 segundos entre tentativas
  • Timeout: 30 segundos por requisição
  • Extraction: Extrai CSV do ZIP automaticamente
  • Seleção de Arquivo:
    1. Procura arquivo contendo "expopm" no nome
    2. Se não encontrar, seleciona o maior arquivo do ZIP
  • Tratamento de Erros:
    • 404: Loga warning e retorna None
    • Outros erros: Tenta novamente até 3 vezes
    • Após 3 tentativas: Retorna None

Código:

def _download_zip(self, url: str) -> bytes:
    """Baixa arquivo zip da URL."""
    max_retries = 3
    retry_count = 0
    
    while retry_count < max_retries:
        try:
            req = requests.get(url, timeout=30)
            
            if req.status_code == 200:
                with ZipFile(io.BytesIO(req.content), "r") as mzip:
                    # Procura arquivo com 'expopm' no nome
                    if any("expopm" in name for name in mzip.namelist()):
                        filename = next(name for name in mzip.namelist() if "expopm" in name)
                        return mzip.read(filename)
                    
                    # Caso contrário, pega o maior arquivo
                    files_sizes = [(info.file_size, info.filename) for info in mzip.infolist()]
                    if files_sizes:
                        largest_file = max(files_sizes, key=lambda x: x[0])[1]
                        return mzip.read(largest_file)
            
            elif req.status_code == 404:
                self.logger.warning(f"Arquivo não encontrado (404): {url}")
                return None
                
        except Exception as e:
            retry_count += 1
            if retry_count < max_retries:
                time.sleep(10)
            else:
                return None
    
    return None

4.5 Método Auxiliar: `_bytes_to_dataframe()`

Converte bytes de CSV em DataFrame pandas.

Processo:

  1. Lê CSV de bytes usando pd.read_csv()
  2. Configurações:
    • Separador: ; (ponto e vírgula)
    • Encoding: latin-1
    • Tipo: dtype=str (todos como string inicialmente)
  3. Converte nomes de colunas para minúsculas
  4. Cria coluna data a partir de mes e año
  5. Formato da data: 01-MM-YYYY convertido para datetime

Código:

def _bytes_to_dataframe(self, csv_bytes: bytes, ie_type: str) -> pd.DataFrame:
    """Converte bytes CSV em DataFrame."""
    df = pd.read_csv(io.BytesIO(csv_bytes), sep=";", encoding="latin-1", dtype=str)
    df.columns = [col.lower() for col in df.columns]
    
    # Criar coluna de data
    df["data"] = df.apply(lambda x: f"01-{x['mes'].zfill(2)}-{x['año']}", axis=1)
    df["data"] = pd.to_datetime(df["data"], format="%d-%m-%Y")
    
    return df

4.6 Método Auxiliar: `_process_ie_type()`

Processa dados de importação ou exportação de forma unificada.

Processo:

  1. Gera lista de anos desde 2020 até o ano atual usando pd.date_range()
  2. Para cada ano:
    • Constrói URL do ZIP usando _make_zip_url()
    • Baixa ZIP usando _download_zip()
    • Se download bem-sucedido, converte para DataFrame usando _bytes_to_dataframe()
    • Adiciona ao conjunto de DataFrames
  3. Concatena todos os DataFrames em um único resultado
  4. Retorna DataFrame consolidado

Tratamento de Erros:

  • Se arquivo não encontrado (404): loga warning e continua com próximo ano
  • Se erro ao baixar: loga erro e continua
  • Se nenhum arquivo processado: retorna DataFrame vazio

Particularidades da Coleta

  • Processamento Multi-Ano: Processa automaticamente todos os anos desde 2020 até o ano atual
  • URLs Fixas: Usa padrão conhecido de nomenclatura para construir URLs
  • Retry Logic: Implementa retry com até 3 tentativas e delay de 10 segundos
  • Extraction Inteligente: Procura arquivo específico ou seleciona maior arquivo do ZIP
  • Encoding Latin-1: Usa encoding Latin-1 (comum em dados argentinos)
  • Criação de Data: Combina colunas mes e año para criar coluna data
  • Colunas Minúsculas: Converte todas as colunas para minúsculas automaticamente

6. Tratamento Específico

6.1 Método `_country_specific_treatment()`

O método implementa tratamento específico para dados da Argentina após a normalização de colunas.

O que faz:

  1. Criação de Data (fallback): Se a coluna data não existir após normalização, mas existirem as colunas año e mes, cria a coluna data combinando essas informações
  2. Formato: Cria data no primeiro dia do mês usando pd.to_datetime() com dicionário

Por que é necessário:

  • Garante que a coluna Data sempre exista, mesmo se houver algum problema na criação anterior
  • Serve como fallback caso a normalização não tenha criado a coluna corretamente

Código:

def _country_specific_treatment(self, df: pd.DataFrame) -> pd.DataFrame:
    """Tratamento específico para dados da Argentina."""
    # Criar coluna data a partir de ano e mês se necessário
    if 'data' not in df.columns and 'año' in df.columns and 'mes' in df.columns:
        df['data'] = pd.to_datetime({
            'year': df['año'].astype(int),
            'month': df['mes'].astype(int),
            'day': 1
        })
    
    return df

6.2 Conversões de Dados

  • Formato de Data: Criada a partir de mes e año no formato 01-MM-YYYY, convertida para datetime usando pd.to_datetime()
  • Formato Numérico: Dados são lidos como string inicialmente (dtype=str) e convertidos posteriormente pelo framework conforme schema do data-contract
  • Encoding: Latin-1 (ISO-8859-1) - comum em dados argentinos
  • Separador CSV: Ponto e vírgula (;)
  • Colunas: Todas convertidas para minúsculas durante processamento

6.3 Filtros Aplicados

Não há filtros específicos aplicados aos dados argentinos. Todos os dados coletados são processados.

10. Particularidades e Observações

Pontos de Atenção

  • Ano Fixo: O crawler processa desde 2020 independente do parâmetro start_date. Este é um comportamento específico deste crawler.
  • Padrão de URLs: O crawler assume que as URLs seguem o padrão conhecido. Se o INDEC mudar o padrão de nomenclatura, será necessário atualizar o método _make_zip_url().
  • Seleção de Arquivo: O crawler procura primeiro por arquivo com "expopm" no nome. Se não encontrar, seleciona o maior arquivo. Isso pode não ser ideal se o ZIP contiver múltiplos arquivos relevantes.
  • Encoding Latin-1: Usa encoding Latin-1 que é comum em dados argentinos, mas pode causar problemas com caracteres especiais se houver.
  • Retry Logic: Implementa retry com delay de 10 segundos. Em caso de muitos erros, o processo pode ser lento.
  • Timeout: Timeout de 30 segundos pode ser insuficiente para arquivos muito grandes.

Diferenças em Relação a Outros Crawlers

  • Arquivos ZIP: Ao contrário de outros crawlers que usam CSV direto, este processa arquivos ZIP.
  • URLs Fixas: Usa padrão conhecido para construir URLs, enquanto outros podem usar web scraping.
  • Retry Logic: Implementa retry explícito com delay, enquanto outros podem usar bibliotecas de retry.
  • Ano Fixo: Processa desde 2020 fixo, enquanto outros podem usar start_date dinamicamente.
  • Encoding Latin-1: Usa Latin-1 ao invés de UTF-8 como padrão.
  • Extraction Inteligente: Seleciona arquivo específico ou maior arquivo do ZIP automaticamente.

Problemas Conhecidos e Soluções

  • Problema: Arquivo não encontrado (404) para determinado ano
    Solução: Crawler continua processando outros anos e loga warning. Normal para anos muito antigos ou muito recentes.
  • Problema: Timeout em downloads grandes
    Solução: Retry logic tenta novamente. Se persistir, pode ser necessário aumentar timeout.
  • Problema: ZIP contém múltiplos arquivos e seleciona o errado
    Solução: Crawler procura primeiro por "expopm". Se necessário, pode ser ajustado para procurar outros padrões.

11. Troubleshooting

11.1 Problemas Comuns

Erro na Coleta: "Arquivo não encontrado (404)"

Possíveis Causas:

  • Ano muito antigo ou muito recente (arquivo ainda não disponível)
  • Padrão de URL mudou no INDEC
  • Portal temporariamente indisponível

Soluções:

  • Verificar manualmente se o arquivo existe no portal
  • Verificar se o padrão de URL ainda está correto
  • Verificar logs para ver quais anos falharam
  • O crawler continua processando outros anos mesmo com alguns 404s

Erro ao Baixar Arquivo: Timeout ou Erro de Conexão

Possíveis Causas:

  • Arquivo muito grande
  • Portal lento ou sobrecarregado
  • Problemas de rede

Soluções:

  • O crawler tenta automaticamente até 3 vezes com delay de 10 segundos
  • Aumentar timeout em _download_zip() se necessário
  • Verificar conectividade com o portal
  • Tentar baixar manualmente para validar disponibilidade

Erro ao Processar CSV: Encoding ou Formato

Possíveis Causas:

  • Encoding diferente de Latin-1
  • Separador diferente de ponto e vírgula
  • Formato do arquivo mudou

Soluções:

  • Verificar encoding real do arquivo
  • Verificar separador usado no CSV
  • Ajustar parâmetros em _bytes_to_dataframe() se necessário
  • Verificar logs para ver erro específico
